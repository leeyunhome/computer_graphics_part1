    [코드 리뷰] 스크린-월드 좌표계 변환 및 포스트 프로세싱(Bloom) 합성 구현

    본 레포지토리의 최근 커밋은 그래픽스 파이프라인에서 매우 중요한 두 가지 개념적 이정표를 달성했습니다.

    첫 번째는 이산적인 픽셀 스크린 공간(Screen Space)을 연속적이고 비율이 보정된 2D 월드 공간(World Space)으로 매핑하여 레이트레이싱의 수학적 기반을 다진 것이며, 두 번째는 이미지 처리의 꽃인 블룸(Bloom) 포스트 프로세싱 효과의 최종 합성 단계(Bright Pass Blur + Original)를 완성한 것입니다.

    1. 핵심 개념: 스크린-월드 좌표계 변환 (Screen-to-World Mapping)

    1.1 변환의 필요성

    컴퓨터 화면은 \(W \times H\) 개의 이산적인 정수 픽셀 그리드로 이루어져 있습니다. 반면, 수학적인 도형(예: 원, 삼각형)과 광선(Ray)은 연속적인 실수 공간에서 정의됩니다. 스크린의 픽셀 좌표를 그대로 사용하여 도형을 그리게 되면 두 가지 문제가 발생합니다. 1. 화면 해상도(\(W, H\))가 바뀔 때마다 도형의 크기와 위치가 물리적으로 왜곡됩니다. 2. 가로세로 비율(Aspect Ratio)이 1:1이 아닌 경우(예: 16:9), 원이 옆으로 찌그러진 타원으로 렌더링됩니다.

    학습자는 이러한 문제를 해결하기 위해 화면 비율이 보정된 월드 좌표계 \([-\text{aspect}, +\text{aspect}] \times [-1, +1]\) 로의 선형 변환(Linear Mapping)을 구현했습니다.

    1.2 수학적 유도

    화면의 종횡비(Aspect Ratio)를 다음과 같이 정의합니다.

    \[\text{aspect} = \frac{\text{width}}{\text{height}}\]

    스크린 공간의 임의의 픽셀 좌표 \((x_s, y_s)\)가 존재할 때, 가로 범위 \(x_s \in [0, \text{width}-1]\)를 \([-\text{aspect}, +\text{aspect}]\)로 매핑하고, 세로 범위 \(y_s \in [0, \text{height}-1]\)를 \([+1, -1]\)로 매핑해야 합니다. (이때, 윈도우 스크린 좌표계는 아래로 갈수록 \(y\)가 증가하지만, 월드 좌표계는 위로 갈수록 \(y\)가 증가하므로 \(y\)축은 반전되어야 합니다.)

    이를 위한 선형 보간 수식은 다음과 같이 유도됩니다.

    X축 매핑 (가로)

    \[x_w = \left( \frac{x_s}{\text{width} - 1} \times 2.0 - 1.0 \right) \times \text{aspect}\]

    이를 코드 레벨에서 연산 효율성을 위해 스케일 오프셋 형태로 정리하면 다음과 같습니다.

    \[x_w = \left( x_s \times \frac{2.0 \cdot \text{aspect}}{\text{width} - 1} \right) - \text{aspect}\]

    Y축 매핑 (세로 - 상하 반전 포함)

    \[y_w = 1.0 - \left( \frac{y_s}{\text{height} - 1} \times 2.0 \right)\]
    \[y_w = -\left( y_s \times \frac{2.0}{\text{height} - 1} - 1.0 \right)\]

    학습자는 이 공식을 정확히 도출하여 TransformScreenToWorld 함수에 반영하였습니다.

    1.3 알고리즘 추상화 (의사코드)

    학습자가 구현한 실시간 도형 렌더링 루프는 다음과 같은 흐름으로 추상화할 수 있습니다.

    # 화면 해상도 정의
width, height = 1280, 720
aspect = width / height

# 매핑에 사용할 스케일 팩터 미리 계산 (프레임당 1회)
xScale = 2.0 * aspect / (width - 1)
yScale = 2.0 / (height - 1)

for y from 0 to height - 1:
    for x from 0 to width - 1:
        # 1. 스크린 좌표를 월드 공간 좌표로 변환
        world_x = (x * xScale) - aspect
        world_y = -((y * yScale) - 1.0)
        world_pos = Vector2(world_x, world_y)

        # 2. 월드 공간에서 수학적 원(Circle) 판별 연산 수행
        if circle.contains(world_pos):
            pixel_buffer[x, y] = circle.color
        else:
            pixel_buffer[x, y] = background_color

    2. 블룸 포스트 프로세싱 합성 (Bloom Composition)

    또 다른 변경점인 Image::Bloom 함수 내의 변경 사항은 추출 및 가우시안 블러 처리가 완료된 밝은 영역 이미지(this->pixels)와 백업해 둔 원본 이미지(pixelsBackup)를 가중치(weight)를 곱해 더하는 최종 합성 공정입니다.

    2.1 합성 공식 및 클램핑

    \[\text{Color}_{\text{final}} = \text{clamp}(\text{Color}_{\text{blurred}} \times \text{weight} + \text{Color}_{\text{original}}, 0.0, 1.0)\]

    포스트 프로세싱 단계에서 색상 값을 누적할 때 가산 혼합(Additive Blending) 특성상 \(1.0\)(흰색)을 초과하는 HDR(High Dynamic Range) 값이 발생할 수 있습니다. 학습자는 이를 최종 LDR 화면에 출력하기 위해 C++17 표준의 std::clamp를 활용하여 \([0.0, 1.0]\) 범위로 안전하게 가두는 예외 처리를 훌륭히 수행했습니다.

    3. 학습자가 직접 구현하며 이해한 핵심 개념

    1. 차원 및 좌표계 변환의 직관: 픽셀 인덱스라는 이산적 영역에서 연속적인 기하학적 영역으로 사고를 확장했습니다. 렌더링 파이프라인 전반에서 쓰이는 NDC(Normalized Device Coordinates)의 개념적 초석을 다졌습니다.
    2. 스크린 공간의 Y축 방향성: 그래픽스 API(DirectX, Vulkan, WebGPU 등) 및 이미지 파일 포맷이 갖는 좌상단(Top-Left) 원점 체계와 수학적 우수 상향(Right-Up) 좌표계 간의 차이를 뒤집기 연산(-1 * (...))을 통해 능숙하게 해결했습니다.
    3. 병렬 제어 및 데이터 흐름: 블룸 필터에서 가우시안 필터의 Separable Convolution(가로, 세로 분할 연산)을 적용해 효율성을 높이고, 최종 합성 단계에서 원본 손실 없이 누적 연산을 수행하는 구조를 확립했습니다.

    4. WebGPU 인터랙티브 데모 시각화 제안

    이 커밋에서 구현한 "화면 비율 보정 스크린-월드 매핑"과 "원 판별 연산"을 WebGPU로 이식하여 브라우저에서 실시간으로 렌더링하면 매우 직관적인 인터랙티브 데모를 만들 수 있습니다.

    // WebGPU WGSL 셰이더 예시 (Fragment Shader)

struct Uniforms {
    canvasSize: vec2<f32>,
    circlePos: vec2<f32>,
    circleRadius: f32,
};

@group(0) @binding(0) var<uniform> ubo: Uniforms;

@fragment
fn fs_main(@builtin(position) fragCoord: vec4<f32>) -> @location(0) vec4<f32> {
    let aspect = ubo.canvasSize.x / ubo.canvasSize.y;

    // 1. 학습자가 구현한 스크린-월드 변환 공식 적용
    let xScale = 2.0 * aspect / (ubo.canvasSize.x - 1.0);
    let yScale = 2.0 / (ubo.canvasSize.y - 1.0);

    let worldX = (fragCoord.x * xScale) - aspect;
    let worldY = -((fragCoord.y * yScale) - 1.0); // WebGPU도 Y축이 아래로 증가하므로 반전
    let worldPos = vec2<f32>(worldX, worldY);

    // 2. 원 내부 판별 (원점으로부터의 거리 측정)
    let dist = distance(worldPos, ubo.circlePos);

    if (dist < ubo.circleRadius) {
        return vec4<f32>(1.0, 0.0, 0.0, 1.0); // 빨간색 원 칠하기
    }

    return vec4<f32>(0.8, 0.8, 0.8, 1.0); // 회색 배경
}

    데모 시각화 동작 모습

    • 실시간 브라우저 렌더링: CPU에서 이중 루프를 돌며 픽셀 버퍼에 복사(memcpy)하던 무거운 작업을 WebGPU의 GPU 병렬 파이프라인(Fragment Shader)을 통해 60FPS 이상의 부드러운 실시간 렌더링으로 전환합니다.
    • 마우스 인터랙션: 사용자가 브라우저 캔버스 위에서 마우스를 움직이면, 마우스의 스크린 좌표 역시 동일한 TransformScreenToWorld 공식을 타고 월드 좌표로 변환되어 원의 중심(ubo.circlePos)을 실시간으로 추적하도록 프로그래밍할 수 있습니다.
    • 종횡비 대응: 브라우저 창 크기를 조절하여 가로세로 비율이 늘어나거나 줄어들어도 원이 타원으로 찌그러지지 않고 완벽한 구형을 유지하는 모습을 시각적으로 즉시 확인할 수 있습니다.
