    BoxBlur5 분리 가능한 컨볼루션(수평 패스) 구현 리뷰

    학습자님, DirectX 11/C++ 그래픽스 스터디 리포지토리에 BoxBlur5 분리 가능한 컨볼루션의 수평 패스를 구현한 커밋을 하셨군요. 매우 잘 하셨습니다! 이미지 처리에서 컨볼루션은 핵심 개념이며, 특히 분리 가능한 컨볼루션은 효율적인 이미지 블러링 기법으로 중요하게 다뤄집니다. 이번 커밋을 통해 학습자님이 어떤 개념들을 이해하고 구현했는지 살펴보겠습니다.

    학습자가 구현하며 이해한 핵심 개념

    이번 커밋은 image.BoxBlur5() 함수를 100번 호출하는 코드를 활성화함으로써, 박스 블러(Box Blur) 필터의 수평 패스 구현을 테스트하는 것으로 보입니다. 이를 통해 학습자님은 다음과 같은 핵심 개념들을 직접 구현하고 이해했을 것이라고 생각합니다.

    1. 컨볼루션(Convolution)의 기본 원리:
      • 이미지의 각 픽셀에 대해 주변 픽셀들의 가중 평균을 계산하여 새로운 픽셀 값을 결정하는 과정입니다. 필터(커널)는 이 가중치를 정의합니다.
    2. 박스 블러(Box Blur) 필터:
      • 주변 픽셀들에 모두 동일한 가중치(예: 1/N)를 적용하여 평균을 내는 가장 간단한 형태의 블러 필터입니다. 5x5 박스 블러의 경우, 각 픽셀은 주변 5x5 영역의 25개 픽셀들의 평균이 됩니다.
    3. 분리 가능한 컨볼루션(Separable Convolution)의 이해:
      • 2D 컨볼루션 커널 \(K\)가 수평 1D 커널 \(K_h\)와 수직 1D 커널 \(K_v\)의 외적(\(K = K_v \otimes K_h\))으로 표현될 수 있을 때, 이를 분리 가능하다고 합니다. 5x5 박스 블러 커널은 다음과 같이 두 개의 1D 커널로 분리될 수 있습니다: $$ K_{5 \times 5} = \begin{pmatrix} 1/5 \ 1/5 \ 1/5 \ 1/5 \ 1/5 \end{pmatrix} \otimes \begin{pmatrix} 1/5 & 1/5 & 1/5 & 1/5 & 1/5 \end{pmatrix} $$
      • 이렇게 분리하면, 원래 \(N \times N\) 커널을 한 번에 적용하는 것(픽셀당 \(N^2\) 연산)보다, 먼저 수평 1D 커널을 적용한 후(픽셀당 \(N\) 연산) 그 결과에 수직 1D 커널을 적용하는 방식(픽셀당 \(N\) 연산)이 훨씬 효율적(\(2N\) 연산)이라는 것을 이해하셨을 것입니다. 5x5 커널의 경우, 25번의 곱셈/덧셈 대신 10번의 곱셈/덧셈으로 동일한 효과를 낼 수 있어 성능 향상에 크게 기여합니다.
    4. 수평 패스(Horizontal Pass) 구현:
      • 분리 가능한 컨볼루션의 첫 단계로, 이미지의 각 행(row)에 대해 수평 1D 커널을 적용하는 과정을 직접 구현하셨습니다. 이는 각 픽셀이 자신의 양옆 픽셀들과 함께 평균되는 과정입니다.
      • 이 과정에서 이미지 경계(boundary)를 벗어나는 픽셀에 대한 처리 방식(예: 클램프(clamp), 반복(repeat), 미러링(mirror) 등)에 대해서도 고민하고 적용했을 것입니다.
    5. 반복 적용의 효과:
      • BoxBlur5() 함수를 100번 반복하여 호출하는 것은 블러 효과를 극대화하는 방법입니다. 여러 번의 박스 블러 반복 적용은 가우시안 블러(Gaussian Blur)와 유사한 부드러운 블러 효과를 낸다고 알려져 있습니다. 이를 통해 블러 강도가 어떻게 조절되는지 직접 확인하셨을 것입니다.

    핵심 알고리즘 (수평 패스)

    박스 블러의 수평 패스는 각 픽셀 \((x, y)\)에 대해 해당 행의 주변 픽셀들을 평균하는 방식으로 작동합니다. 5x5 박스 블러의 수평 1D 커널은 [1/5, 1/5, 1/5, 1/5, 1/5]입니다.

    입력 이미지 \(I\)와 출력될 중간 이미지 \(I_{temp}\)가 있을 때, 각 픽셀의 새로운 값 \(I_{temp}[x, y]\)는 다음과 같이 계산됩니다:

    함수 BoxBlurHorizontal(SourceImage, TargetImage, KernelRadius):
  ImageWidth = SourceImage.Width
  ImageHeight = SourceImage.Height
  KernelSize = 2 * KernelRadius + 1 // 예: KernelRadius=2 이면 KernelSize=5

  // 이미지의 각 행(row)에 대해 반복
  for y from 0 to ImageHeight - 1:
    // 각 행의 각 픽셀(column)에 대해 반복
    for x from 0 to ImageWidth - 1:
      Sum = (0, 0, 0) // RGB 채널 또는 각 채널별로 합계 초기화

      // 5x5 박스 블러의 경우 KernelRadius = 2 (중심 픽셀 포함 총 5개)
      // 현재 픽셀을 중심으로 좌우로 KernelRadius만큼 떨어진 픽셀들을 합산
      for i from -KernelRadius to KernelRadius:
        // 경계 조건 처리: 이미지 범위를 벗어나지 않도록 픽셀 좌표 클램프
        NeighborX = Clamp(x + i, 0, ImageWidth - 1) 
        Sum += SourceImage.GetPixel(NeighborX, y)

      // 계산된 합계를 커널 크기로 나누어 평균을 구하고 결과 이미지에 저장
      TargetImage.SetPixel(x, y, Sum / KernelSize)

  return TargetImage

    수식으로 표현하면, \(P'_{x,y}\)는 원본 이미지 \(I\)의 픽셀 \((x,y)\)에 대한 수평 블러링 결과이며, \(K_h\)는 5-탭 수평 박스 블러 커널 \([1/5, 1/5, 1/5, 1/5, 1/5]\)입니다. $$ P'{x,y} = \sum $$ 여기서 }^{2} I_{clamp(x+i), y} \cdot K_{h_i} = \frac{1}{5} \sum_{i = -2}^{2} I_{clamp(x+i), y\(clamp(v, min, max)\)는 값이 \(min\)과 \(max\) 사이에 있도록 제한하는 함수입니다. 예를 들어, \(0\) 미만이면 \(0\)으로, 너비(\(ImageWidth-1\))보다 크면 너비(\(ImageWidth-1\))로 조정됩니다.

    WebGPU 인터랙티브 데모

    이 박스 블러의 수평 패스를 WebGPU의 Compute Shader를 이용해 구현한다면 다음과 같은 방식으로 작동하며 시각화될 것입니다.

    1. Compute Shader 작성: GLSL 또는 WGSL로 작성된 Compute Shader는 각 워크그룹(workgroup) 또는 개별 인보케이션(invocation)이 이미지의 특정 픽셀을 처리하도록 구성됩니다.
    2. 데이터 바인딩: 원본 이미지 데이터는 storage_texture_2d<rgba8unorm, read>와 같은 텍스처 형태로 바인딩되고, 수평 블러링된 결과가 저장될 중간 이미지 텍스처는 storage_texture_2d<rgba8unorm, write>로 바인딩됩니다.
    3. 커널 실행: Compute Shader의 각 인보케이션은 global_invocation_id.xy를 통해 처리할 픽셀의 전역 좌표 \((x, y)\)를 얻습니다.
    4. 픽셀 처리: 각 인보케이션은 자신의 \((x, y)\) 좌표를 중심으로 좌우로 2칸씩 떨어진 픽셀들 (총 5개)의 색상 값을 원본 이미지 텍스처에서 읽어옵니다. 이 5개 픽셀의 색상 값을 평균하여, 결과 텍스처의 \((x, y)\) 위치에 기록합니다. 경계 조건 처리는 textureLoad 함수와 함께 clamp 등의 유틸리티 함수를 사용하거나, textureSample의 샘플링 모드를 clamp_to_edge로 설정하여 간단히 처리할 수 있습니다.
    5. 시각적 효과: 브라우저에서 이 데모를 실행하면, 원본 이미지가 로드된 후 컴퓨트 셰이더가 실행되어 새로운 이미지로 렌더링될 것입니다. 시각적으로는 이미지가 마치 좌우로 길게 늘어트린 것처럼 수평 방향으로만 흐려지는(smearing) 효과를 관찰할 수 있습니다. 수직 방향의 디테일은 그대로 유지되지만, 수평 방향의 디테일은 크게 손실되어 뿌옇게 보일 것입니다. BoxBlur5() 함수를 100번 반복 적용한 결과라면, 이미지는 거의 완전히 수평 방향으로만 색상 띠가 이어진 것처럼 보여 원본 이미지의 형체를 알아보기 어려울 정도로 강한 수평 블러가 적용될 것입니다.
