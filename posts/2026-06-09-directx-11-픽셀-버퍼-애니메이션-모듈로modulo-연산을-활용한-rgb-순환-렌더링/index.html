    [DirectX 11] 픽셀 버퍼 애니메이션: 모듈로(Modulo) 연산을 활용한 RGB 순환 렌더링

    컴퓨터 그래픽스를 공부하면서 가장 처음 마주하게 되는 흥미로운 순간은, 화면을 구성하는 수많은 픽셀(Pixel)들을 내 마음대로 제어할 수 있게 되는 때입니다.

    이번 포스팅에서는 DirectX 11 초기화 및 ImGui 세팅을 마친 환경에서, CPU 상의 픽셀 버퍼를 직접 조작하여 RGB 색상이 순환하는 픽셀 애니메이션을 구현한 과정과 그 원리를 정리해 보았습니다.

    1. 2D 캔버스와 1차원 픽셀 버퍼의 이해

    우리가 보는 모니터 화면은 2차원(2D) 격자 구조를 가지지만, 컴퓨터 메모리 상에서 이 데이터는 일반적으로 1차원 배열(1D Array)로 연속적으로 할당됩니다.

    화면의 너비가 \(W\)이고 높이가 \(H\)일 때, 전체 픽셀의 개수는 \(W \times H\)가 됩니다. 이 1차원 배열에서 특정 픽셀의 인덱스 \(i\)는 순차적으로 증가하며, 특정 좌표 \((x, y)\)의 픽셀에 접근하기 위한 수학적 변환은 다음과 같습니다.

    \[ Index = y \times W + x \]

    이번 구현에서는 전체 캔버스를 회색 바탕(0.8, 0.8, 0.8)으로 초기화한 뒤, 매 프레임마다 인덱스 \(i\)를 증가시키며 픽셀의 색상을 변경하는 방식을 사용했습니다.

    2. 모듈로(Modulo) 연산을 활용한 색상 순환 (RGB Cycling)

    애니메이션의 핵심은 단순히 하나의 색상으로 픽셀을 채우는 것이 아니라, Red, Green, Blue 세 가지 색상이 순차적으로 반복되도록 만드는 것입니다. 이를 위해 수학의 모듈로(Modulo, 나머지) 연산을 활용했습니다.

    현재 칠해야 할 픽셀의 인덱스 \(i\)를 3으로 나눈 나머지 값에 따라 색상을 결정합니다. 이를 수식으로 표현하면 다음과 같습니다.

    \[ f(i) = i \pmod 3 \]

    이 함수 \(f(i)\)의 결과값은 항상 0, 1, 2 중 하나가 되며, 이를 4차원 벡터인 RGBA 색상값에 매핑합니다.

    • \(f(i) = 0\) 인 경우: 빨간색 (1.0, 0.0, 0.0, 1.0)
    • \(f(i) = 1\) 인 경우: 초록색 (0.0, 1.0, 0.0, 1.0)
    • \(f(i) = 2\) 인 경우: 파란색 (0.0, 0.0, 1.0, 1.0)

    이 간단한 수학적 규칙 덕분에, 인덱스가 증가함에 따라 화면에는 빨강-초록-파랑 순서의 패턴이 아름답게 그려지게 됩니다.

    3. 프레임 페이싱과 시각적 디버깅 (Frame Pacing)

    그래픽스 렌더링 루프(Update Loop)는 컴퓨터의 성능에 따라 1초에 수백 번에서 수천 번 이상 매우 빠르게 실행됩니다. 만약 아무런 제어 없이 픽셀 인덱스를 증가시키면, 애니메이션이 너무 순식간에 끝나버려 사람의 눈으로 색상 변화의 과정을 관찰하기 어렵습니다.

    이를 해결하기 위해 업데이트 루프 내부에 의도적인 지연(Delay)을 추가했습니다.

    • 300ms 지연 (Sleep): 한 프레임이 업데이트될 때마다 0.3초(300ms)씩 스레드를 대기시킵니다.
    • 목적: 픽셀이 하나씩 순차적으로 RGB 색상으로 변하는 과정을 시각적으로 뚜렷하게 관찰하고 검증(Debugging)하기 위함입니다.

    4. 개념적 구현 (Pseudo-code)

    실제 코드를 추상화한 의사 코드(Pseudo-code)는 다음과 같습니다. C++의 std::vector와 삼항 연산자를 활용하여 간결하게 표현할 수 있습니다.

    // Update Loop 내부 로직 (매 프레임 호출됨)
void UpdateFrame() 
{
    // 1. 시각적 관찰을 위한 300ms 지연
    PauseThread(300);

    // 2. 현재 업데이트할 픽셀 인덱스 (정적 변수 활용)
    static int currentIndex = 0;

    // 3. 전체 캔버스 픽셀을 기본색상(회색)으로 초기화
    Vector4D defaultColor = {0.8, 0.8, 0.8, 1.0};
    FillBuffer(defaultColor);

    // 4. 모듈로 연산을 통한 RGB 색상 결정
    Vector4D targetColor;
    if (currentIndex % 3 == 0)      targetColor = {1.0, 0.0, 0.0, 1.0}; // Red
    else if (currentIndex % 3 == 1) targetColor = {0.0, 1.0, 0.0, 1.0}; // Green
    else                            targetColor = {0.0, 0.0, 1.0, 1.0}; // Blue

    // 5. 픽셀 버퍼에 색상 적용
    PixelBuffer[currentIndex] = targetColor;

    // 6. 인덱스 증가 및 캔버스 크기 초과 방지
    currentIndex++;
    if (currentIndex >= TotalPixels) {
        // 루프 초기화 또는 종료 처리
    }
}

    (참고: 위 코드는 알고리즘의 원리를 설명하기 위해 작성된 개념적 코드입니다.)

    💡 이번 커밋을 통해 배운 점 및 향후 과제

    1. 메모리 조작의 직관성: CPU 레벨에서 1차원 배열의 인덱스 값을 조작하여 2D 화면의 픽셀을 직접 그려내는 원리를 완벽하게 이해할 수 있었습니다.
    2. 수학적 패턴의 활용: 조건문(if-else)을 길게 나열하는 대신, 나머지 연산(\(\pmod\))과 삼항 연산자를 결합하여 매우 우아하고 효율적인 코드를 작성하는 방법을 연습했습니다.
    3. 한계점과 개선 방향 (Delta Time의 필요성): 이번에는 애니메이션 속도를 조절하기 위해 하드웨어적인 Sleep() 함수를 사용했습니다. 하지만 실제 게임 엔진이나 전문적인 그래픽스 파이프라인에서는 전체 시스템을 멈추게 하는 Sleep 대신, 이전 프레임과 현재 프레임 사이의 시간차인 델타 타임(\(\Delta t\), Delta Time)을 계산하여 프레임 레이트에 독립적인(Framerate-independent) 애니메이션을 구현해야 합니다. 이는 다음 학습 단계에서 적용해 볼 예정입니다.
