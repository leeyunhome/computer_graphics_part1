    DirectX 11/C++ 그래픽스 스터디: 첫 커밋 분석

    학습자님, DirectX 11/C++ 그래픽스 스터디 레포지토리에 첫 커밋을 완료하신 것을 축하드립니다! "first commit"이라는 메시지는 이 프로젝트의 성공적인 첫걸음을 의미하며, 대개 그래픽스 애플리케이션의 핵심 기반을 다지는 과정에서 이루어집니다. 코드 변경사항이 명시되어 있지 않지만, 일반적으로 DirectX 11 프로젝트의 초기 설정은 다음과 같은 주요 개념들을 직접 구현하고 이해하는 과정을 포함합니다.

    학습자가 직접 구현하며 이해한 핵심 개념

    첫 커밋에서는 주로 그래픽스 애플리케이션이 실행될 윈도우 환경을 설정하고, DirectX 11 렌더링을 위한 기본 파이프라인을 초기화하는 데 중점을 둡니다.

    1. WinAPI 기반 윈도우 생성 및 관리

    • 개념 이해: 그래픽스 애플리케이션은 운영체제(Windows)가 제공하는 윈도우 위에서 렌더링됩니다. 학습자는 WinMain 함수를 진입점으로 사용하여 WNDCLASSEX 구조체를 정의하고 RegisterClassEx로 윈도우 클래스를 등록합니다. 이후 CreateWindowEx 함수를 호출하여 실제 윈도우를 생성하고, ShowWindow, UpdateWindow로 화면에 표시합니다.
    • 구현 과정:
      • 윈도우 클래스 속성(아이콘, 커서, 배경색, WndProc 콜백 함수 등)을 설정합니다.
      • WndProc 함수를 통해 윈도우로 들어오는 메시지(예: 마우스 클릭, 키보드 입력, 윈도우 크기 변경)를 처리하는 방법을 학습합니다.
      • GetMessage, TranslateMessage, DispatchMessage로 구성된 메시지 루프를 구현하여 윈도우의 이벤트 처리가 실시간으로 이루어지도록 합니다.
    • 학습 의미: 그래픽스 렌더링은 이 윈도우 클라이언트 영역 내에서 이루어지므로, 윈도우 자체의 생성 및 생명주기 관리는 렌더링의 가장 기본적인 기반이 됩니다.

    2. DirectX 디바이스 및 스왑 체인 초기화

    • 개념 이해: DirectX 11을 사용하여 GPU와 통신하려면 먼저 ID3D11Device와 ID3D11DeviceContext를 생성해야 합니다. 또한, 렌더링 결과를 화면에 표시하기 위해서는 IDXGISwapChain이라는 특별한 객체가 필요합니다.
    • 핵심 함수: D3D11CreateDeviceAndSwapChain 함수가 이 세 가지 핵심 객체를 한 번에 생성합니다.
    • 구현 과정:
      • D3D_DRIVER_TYPE_HARDWARE를 지정하여 GPU 하드웨어 가속을 사용하도록 설정합니다.
      • D3D_FEATURE_LEVEL_11_0을 통해 DirectX 11의 모든 기능을 사용하겠다고 선언합니다.
      • DXGI_SWAP_CHAIN_DESC 구조체를 채워 스왑 체인의 속성(예: 백 버퍼 개수, 이미지 포맷 DXGI_FORMAT_R8G8B8A8_UNORM, 윈도우 핸들 hWnd, 멀티샘플링 설정 등)을 정의합니다. 특히, BufferDesc의 Width와 Height는 윈도우 클라이언트 영역의 크기와 일치시킵니다.
      • ID3D11Device: GPU의 리소스를 생성하고 관리하는 역할을 합니다.
      • ID3D11DeviceContext: 렌더링 명령(그리기, 셰이더 설정 등)을 GPU에 보내는 역할을 합니다.
      • IDXGISwapChain: 렌더링된 이미지를 담는 백 버퍼(Back Buffer)와 화면에 표시되는 프론트 버퍼(Front Buffer) 사이의 교환(swap)을 관리합니다.
    • 수식(개념적): 렌더링 파이프라인의 핵심 초기화 과정은 다음과 같이 추상화할 수 있습니다. $$ \text{DeviceAndSwapChain} = \text{D3D11CreateDeviceAndSwapChain}(\text{GPU_Hardware}, \text{DX11_Features}, \text{SwapChain_Config}) $$
    • 학습 의미: 이 과정은 WinAPI 윈도우와 GPU를 연결하는 다리 역할을 합니다. 학습자는 그래픽스 카드의 종류, DirectX 버전, 렌더링 방식(창 모드/전체 화면) 등에 따라 D3D11CreateDeviceAndSwapChain 함수의 인자들이 어떻게 달라지는지 이해하게 됩니다.

    3. 렌더 타겟 뷰(Render Target View) 설정

    • 개념 이해: GPU가 실제로 픽셀을 그릴 대상(타겟)을 지정하는 것이 렌더 타겟 뷰입니다. 스왑 체인에서 제공하는 백 버퍼를 이 렌더 타겟으로 사용합니다.
    • 구현 과정:
      • IDXGISwapChain::GetBuffer를 호출하여 스왑 체인의 첫 번째 백 버퍼(실제로는 2D 텍스처)에 대한 ID3D11Texture2D 포인터를 얻습니다.
      • ID3D11Device::CreateRenderTargetView 함수를 사용하여 이 ID3D11Texture2D로부터 렌더 타겟 뷰(ID3D11RenderTargetView)를 생성합니다.
      • ID3D11DeviceContext::OMSetRenderTargets를 호출하여 이 렌더 타겟 뷰를 Output Merger 스테이지에 바인딩합니다. 이제 모든 렌더링 명령은 이 뷰가 가리키는 백 버퍼에 그려집니다.
    • 학습 의미: 렌더링 결과가 어디에 저장되어야 하는지 명확히 지정함으로써, GPU가 올바른 메모리 위치에 픽셀 데이터를 작성할 수 있도록 합니다.

    4. 뷰포트(Viewport) 설정

    • 개념 이해: 뷰포트는 렌더 타겟(백 버퍼) 내에서 실제로 렌더링이 이루어질 직사각형 영역을 정의합니다. 이는 정규화된 디바이스 좌표(NDC, Normalized Device Coordinates)를 화면상의 픽셀 좌표로 변환하는 데 사용됩니다.
    • 구현 과정:
      • D3D11_VIEWPORT 구조체를 정의하여 뷰포트의 TopLeftX, TopLeftY, Width, Height를 설정합니다. 일반적으로 윈도우의 클라이언트 영역 전체를 뷰포트로 사용합니다. MinDepth와 MaxDepth는 3D 렌더링 시 깊이 범위에 사용됩니다.
      • ID3D11DeviceContext::RSSetViewports 함수를 호출하여 뷰포트를 Rasterizer 스테이지에 바인딩합니다.
    • 수식(개념적): 뷰포트 변환은 다음과 같은 스케일링 및 오프셋 변환으로 이해할 수 있습니다. $$ x_{screen} = \frac{\text{Width}}{2} \cdot x_{NDC} + \text{TopLeftX} + \frac{\text{Width}}{2} \ y_{screen} = -\frac{\text{Height}}{2} \cdot y_{NDC} + \text{TopLeftY} + \frac{\text{Height}}{2} \ z_{screen} = \text{MinDepth} + (\text{MaxDepth} - \text{MinDepth}) \cdot z_{NDC} $$
    • 학습 의미: 뷰포트를 통해 렌더링 결과가 렌더 타겟의 어느 부분에 표시될지 제어할 수 있습니다. 예를 들어, 한 윈도우에 여러 개의 작은 화면(미니맵 등)을 그릴 때 유용하게 사용됩니다.

    5. 프레임 렌더링 루프의 기본 구조 (화면 지우기)

    • 개념 이해: 게임이나 그래픽스 애플리케이션은 지속적으로 화면을 업데이트해야 합니다. 이를 위해 "프레임 루프"라는 것이 필요하며, 각 프레임마다 이전 내용을 지우고 새로운 내용을 그리는 과정을 반복합니다.
    • 구현 과정:
      • 메시지 루프 내 또는 별도의 함수에서 매 프레임마다 호출되는 렌더링 함수를 만듭니다.
      • ID3D11DeviceContext::ClearRenderTargetView: 렌더링을 시작하기 전에 이전에 그려진 내용을 지우고 특정 색상으로 렌더 타겟을 채웁니다. 이는 마치 그림판에서 새 도화지를 여는 것과 같습니다.
      • IDXGISwapChain::Present: 백 버퍼에 그려진 최종 이미지를 프론트 버퍼로 "전송"하여 사용자 화면에 표시합니다. SyncInterval 파라미터로 수직 동기화(V-Sync)를 제어할 수 있습니다.
    • 학습 의미: 이 루프는 그래픽스 애플리케이션의 심장과 같습니다. ClearRenderTargetView와 Present는 한 프레임의 가장 기본적인 시작과 끝을 담당하며, 앞으로 그려질 모든 3D 오브젝트들은 이 두 함수 사이에 위치하게 됩니다.

    6. 자원 해제

    • 개념 이해: DirectX 객체들은 COM(Component Object Model) 객체이며, 사용 후에는 반드시 Release() 메서드를 호출하여 참조 카운트를 줄이고 자원을 해제해야 합니다. 그렇지 않으면 메모리 누수나 GPU 자원 누수가 발생할 수 있습니다.
    • 구현 과정: 프로그램 종료 시 생성했던 모든 ID3D11Device, ID3D11DeviceContext, IDXGISwapChain, ID3D11RenderTargetView 등의 객체에 대해 Release()를 호출합니다.
    • 학습 의미: 올바른 자원 관리는 안정적이고 효율적인 그래픽스 애플리케이션 개발의 기본입니다.

    WebGPU 인터랙티브 데모

    위에서 설명한 DirectX 11의 초기화 및 화면 지우기 개념들은 WebGPU에서도 유사하게 적용될 수 있습니다. WebGPU는 브라우저 환경에서 GPU 자원에 접근할 수 있게 해주는 최신 API로, DirectX 11의 많은 개념들이 추상화되어 있지만 본질은 동일합니다.

    WebGPU로 시각화되는 모습:

    DirectX 11의 "first commit"이 백 버퍼를 특정 색상으로 지우고 화면에 표시하는 것이라면, WebGPU 데모는 HTML <canvas> 요소를 특정 색상으로 클리어하는 모습을 보여줄 수 있습니다.

    1. 어댑터 및 디바이스 요청 (Request Adapter & Device): DirectX의 D3D11CreateDeviceAndSwapChain과 유사하게, navigator.gpu.requestAdapter()와 adapter.requestDevice()를 통해 브라우저가 접근할 수 있는 GPU와 논리적인 디바이스를 얻습니다. 이는 GPU와의 통신 채널을 여는 과정입니다.

    2. 캔버스 컨텍스트 구성 (Configure Canvas Context): HTML 캔버스를 렌더링 대상으로 사용하기 위해 canvas.getContext("webgpu")를 호출하고, context.configure()를 통해 스왑 체인(swap chain)과 유사한 역할을 하는 캔버스 컨텍스트를 구성합니다. 여기서 픽셀 포맷(예: bgra8unorm)을 지정하고, DirectX의 DXGI_SWAP_CHAIN_DESC에서 BufferDesc를 설정하는 것과 유사합니다.

    3. 렌더 패스 인코더 생성 및 화면 지우기 (Create Render Pass Encoder & Clear): DirectX의 ID3D11DeviceContext::ClearRenderTargetView와 OMSetRenderTargets는 WebGPU에서 device.createCommandEncoder()로 커맨드 인코더를 얻고, encoder.beginRenderPass()를 통해 렌더 패스를 시작하면서 colorAttachments 배열의 loadOp를 'clear'로, clearValue를 특정 색상(예: [0.0, 0.2, 0.4, 1.0] - 어두운 파란색)으로 설정하는 것으로 대체됩니다. 이 과정을 통해 GPU는 해당 캔버스 영역을 지정된 색상으로 채우게 됩니다.

    4. 명령 제출 (Submit Commands): renderPass.end()로 렌더 패스를 종료하고, commandEncoder.finish()로 커맨드 버퍼를 완성한 후, device.queue.submit([commandBuffer])를 통해 GPU에 명령을 제출하면, 캔버스는 즉시 지정된 색상으로 업데이트됩니다. 이는 DirectX의 IDXGISwapChain::Present와 유사하게, 백 버퍼의 내용을 화면에 표시하는 과정입니다.

    인터랙티브 요소: 사용자가 버튼을 클릭할 때마다 clearValue를 다른 색상으로 변경하여 캔버스를 지우도록 함으로써, 렌더링 루프의 "화면 지우기" 단계가 실시간으로 어떻게 작동하는지 시각적으로 확인할 수 있습니다. 예를 들어, 버튼을 누를 때마다 배경색이 파란색 -> 녹색 -> 빨간색 등으로 바뀌는 모습은 ClearRenderTargetView의 효과를 직관적으로 보여줍니다.

    다시 한번 첫 커밋을 축하드리며, 앞으로 DirectX 11의 더 깊은 개념들을 익히면서 멋진 그래픽스 프로젝트를 만들어 나가시길 기대합니다!
