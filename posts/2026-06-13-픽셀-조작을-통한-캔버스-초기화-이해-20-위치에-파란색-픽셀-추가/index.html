    픽셀 조작을 통한 캔버스 초기화 이해: (2,0) 위치에 파란색 픽셀 추가

    학습자님, DirectX 11 스터디 레포지토리에 커밋하신 내용을 잘 살펴보았습니다. (2,0) 위치에 파란색 픽셀을 추가하는 변경사항은 그래픽스 초기화 과정에서 매우 중요한 여러 개념을 직접 구현하고 이해했음을 보여줍니다.

    이번 커밋을 통해 학습자님이 깊이 이해했을 핵심 개념들은 다음과 같습니다.

    1. 2D 캔버스 좌표를 1D 배열 인덱스로 매핑하는 방법

    가장 먼저 눈에 띄는 것은 2D 공간의 픽셀 위치 (x, y)를 std::vector<Vec4> pixels와 같은 1차원 배열의 인덱스로 정확히 변환하는 방법입니다. 캔버스는 너비(width)와 높이(height)를 가진 2차원 그리드이지만, 메모리 상에서는 대부분 1차원 배열로 평탄화되어 저장됩니다.

    학습자님은 pixels[2 + canvasWidth * 0]와 같이 (x, y) 좌표를 1차원 인덱스로 변환하는 공식을 성공적으로 적용했습니다. 여기서 y가 0이므로 pixels[2]로 간단해지며, 이는 캔버스의 첫 번째 행에 세 번째 픽셀을 의미합니다.

    일반적인 2D 좌표 \((x, y)\)를 1D 배열 인덱스 \(i\)로 변환하는 공식은 다음과 같습니다:

    \[ i = x + \text{canvasWidth} \times y \]

    이 공식을 이해하고 활용하는 것은 텍스처 데이터, 이미지 처리, GPGPU 연산 등 컴퓨터 그래픽스의 다양한 분야에서 매우 기초적이면서도 핵심적인 기술입니다.

    2. RGBA 색상 표현 및 조작

    Vec4{ 0.0f, 0.0f, 1.0f, 1.0f } 코드를 통해 학습자님은 RGBA(Red, Green, Blue, Alpha) 색상 모델을 정확히 이해하고 있음을 보여주었습니다. 각 구성 요소는 0.0에서 1.0 사이의 부동 소수점 값으로 표현되며, 각각 빨강, 초록, 파랑, 그리고 불투명도를 나타냅니다.

    • 0.0f: 해당 색상 채널이 완전히 없는 상태 (어두움)
    • 1.0f: 해당 색상 채널이 완전히 있는 상태 (밝음)

    따라서 (0.0f, 0.0f, 1.0f, 1.0f)는 빨강과 초록이 없고 파랑만 최대치로 채워진 '불투명한 파란색'을 의미합니다. 이미 앞서 빨간색(1.0f, 0.0f, 0.0f, 1.0f)과 노란색(1.0f, 1.0f, 0.0f, 1.0f)을 설정한 것을 통해 이 개념을 완벽히 숙지하고 있음을 알 수 있습니다.

    3. 텍스처 데이터 초기화 및 업데이트 파이프라인 (CPU-GPU 데이터 전송의 시작)

    이 코드 변경은 pixels라는 CPU 메모리 상의 벡터 데이터를 수정하는 것입니다. 이 pixels 데이터는 이후 D3D11_MAPPED_SUBRESOURCE와 같은 DirectX API를 통해 GPU 메모리(텍스처)로 전송되어 화면에 실제 픽셀로 렌더링될 것입니다.

    학습자님은 텍스처가 화면에 표시되기 전, 그 초기 데이터를 CPU 상에서 어떻게 준비하고 조작하는지를 직접 경험한 것입니다. 이는 CPU에서 생성된 데이터를 GPU로 넘겨주는 그래픽스 파이프라인의 첫 단계를 이해하는 데 매우 중요합니다.

    의사 코드로 표현하면 다음과 같습니다:

    함수 캔버스_초기화():
    캔버스_너비 = (초기화 값)
    캔버스_높이 = (초기화 값)

    // 1. CPU 메모리에 픽셀 데이터 배열 생성 및 초기화
    픽셀_배열 = 크기가 (캔버스_너비 * 캔버스_높이)인 Vec4 벡터
    모든 픽셀을 기본 회색(0.8f, 0.8f, 0.8f, 1.0f)으로 초기화

    // 2. 특정 위치의 픽셀 색상 변경
    // (0,0) 위치에 빨간색 설정
    픽셀_배열[0 + 캔버스_너비 * 0] = {1.0f, 0.0f, 0.0f, 1.0f} 
    // (1,0) 위치에 노란색 설정
    픽셀_배열[1 + 캔버스_너비 * 0] = {1.0f, 1.0f, 0.0f, 1.0f}
    // (2,0) 위치에 파란색 설정 (이번 커밋 변경사항)
    픽셀_배열[2 + 캔버스_너비 * 0] = {0.0f, 0.0f, 1.0f, 1.0f}

    // 3. (다음 단계: DirectX API를 사용하여) 이 픽셀 데이터를 GPU 텍스처로 업로드
    // D3D11_MAPPED_SUBRESOURCE 구조체를 사용하여 텍스처 자원에 접근
    // 픽셀_배열 데이터를 GPU 텍스처 메모리로 복사
    // 텍스처 자원 언맵

    WebGPU 인터랙티브 데모

    이번 학습자님의 변경사항을 WebGPU Compute Shader로 시각화한다면, 픽셀 데이터 생성 로직을 CPU에서 GPU로 옮겨 연산하는 방식을 보여줄 수 있습니다.

    시나리오: 브라우저에서 실행되는 WebGPU 애플리케이션이 캔버스 크기의 storage_texture (쓰기 가능한 텍스처)를 하나 생성합니다. 이 텍스처는 초기에는 아무런 색상 정보가 없습니다.

    Compute Shader 동작: Compute Shader는 캔버스 픽셀 수만큼의 스레드를 디스패치합니다. 각 스레드는 고유한 global_invocation_id를 통해 자신의 (x, y) 픽셀 좌표를 알 수 있습니다. 이 (x, y) 좌표를 기반으로 각 스레드는 다음과 같은 로직을 수행합니다:

    • 만약 (x, y)가 (0, 0)이면, 해당 픽셀에 빨간색(vec4<f32>(1.0, 0.0, 0.0, 1.0))을 할당합니다.
    • 만약 (x, y)가 (1, 0)이면, 해당 픽셀에 노란색(vec4<f32>(1.0, 1.0, 0.0, 1.0))을 할당합니다.
    • 만약 (x, y)가 (2, 0)이면, 해당 픽셀에 파란색(vec4<f32>(0.0, 0.0, 1.0, 1.0))을 할당합니다.
    • 그 외의 모든 픽셀에는 기본 회색(vec4<f32>(0.8, 0.8, 0.8, 1.0))을 할당합니다.

    이 Compute Shader가 실행을 완료하면, storage_texture에는 학습자님이 CPU에서 정의한 것과 동일한 패턴의 픽셀 데이터가 GPU 상에 생성됩니다. 이후 이 텍스처는 화면에 렌더링되어 첫 번째 행에 빨강, 노랑, 파랑 픽셀이 나란히 표시되고 나머지 영역은 회색으로 채워진 이미지를 볼 수 있을 것입니다. 이는 CPU가 아닌 GPU에서 병렬적으로 픽셀 데이터를 생성하는 강력한 방법을 보여주는 예시가 됩니다.
