    광선-구체 교차 판정과 법선 벡터 계산의 수학적 구현

    본 커밋은 레이트레이싱(Raytracing)의 핵심이자 첫 관문인 광선-구체 교차 판정(Ray-Sphere Intersection)과, 충돌 지점에서의 법선 벡터(Normal Vector)를 계산하여 3D 형태를 정의하는 단계를 구현한 것입니다.

    단순히 화면에 원을 그리는 것을 넘어, 수학적 방정식을 통해 3차원 공간 상의 구체와 직선의 교점을 찾고 3차원 기하 정보(위치, 법선)를 추출하는 고전적이면서도 매우 중요한 그래픽스 알고리즘이 적용되었습니다.

    1. 수학적 원리 및 핵심 개념

    이 커밋의 핵심은 광선의 방정식과 구체의 방정식을 연립하여 교점을 구하는 것입니다.

    광선(Ray)의 방정식

    광선은 시작점 \(\mathbf{O}\)와 방향 벡터 \(\mathbf{D}\)로 정의되며, 매개변수 \(t\) (단, \(t \ge 0\))에 대한 함수로 나타낼 수 있습니다. $$ \mathbf{P}(t) = \mathbf{O} + t\mathbf{D} $$

    구체(Sphere)의 방정식

    중심이 \(\mathbf{C}\)이고 반지름이 \(r\)인 구 위의 임의의 점 \(\mathbf{P}\)는 다음 조건을 만족합니다. $$ |\mathbf{P} - \mathbf{C}|^2 = r^2 \implies (\mathbf{P} - \mathbf{C}) \cdot (\mathbf{P} - \mathbf{C}) - r^2 = 0 $$

    연립 및 판별식 유도

    광선이 구체와 만나는 점을 찾기 위해 구체의 방정식에 광선의 방정식을 대입합니다. 편의상 \(\mathbf{V} = \mathbf{O} - \mathbf{C}\)로 치환하면 다음과 같은 \(t\)에 대한 2차 방정식이 유도됩니다. $$ (\mathbf{D} \cdot \mathbf{D})t^2 + 2(\mathbf{D} \cdot \mathbf{V})t + (\mathbf{V} \cdot \mathbf{V}) - r^2 = 0 $$

    여기서 광선의 방향 벡터 \(\mathbf{D}\)가 단위 벡터(Unit Vector)라고 가정하면, \(\mathbf{D} \cdot \mathbf{D} = 1\)이 되어 이차방정식의 계수 \(a\)는 \(1\)이 됩니다. 따라서 짝수 판별식(Discriminant, 코드 내 nabla로 표현)을 적용하여 근의 공식으로 \(t\)를 구할 수 있습니다.

    \[ \frac{D}{4} = (\mathbf{D} \cdot \mathbf{V})^2 - ((\mathbf{V} \cdot \mathbf{V}) - r^2) \]
    • \(\frac{D}{4} > 0\): 광선이 구체를 관통함 (두 개의 교점)
    • \(\frac{D}{4} = 0\): 광선이 구체에 접함 (한 개의 교점)
    • \(\frac{D}{4} < 0\): 광선이 구체를 비껴감 (교점 없음)

    2. 핵심 알고리즘 구조 (의사코드)

    학습자가 작성한 Sphere.h 파일의 충돌 함수는 구조적으로 다음과 같이 추상화할 수 있습니다.

    function IntersectRayCollision(ray) {
    Initialize hit with negative distance (default: no collision)

    Let V = ray.start - sphere.center
    Let b = 2.0 * dot(ray.dir, V)
    Let c = dot(V, V) - sphere.radius^2

    // 짝수 판별식 계산
    Let discriminant = (b / 2.0)^2 - c

    if discriminant >= 0.0 then
        // 두 개의 교점(거리) 계산
        Let d1 = -b / 2.0 + sqrt(discriminant)
        Let d2 = -b / 2.0 - sqrt(discriminant)

        // 더 가까운 교점을 선택
        hit.distance = min(d1, d2)

        // 충돌 지점 계산: P(t) = O + tD
        hit.point = ray.start + ray.dir * hit.distance

        // 법선 벡터 계산: (P - C)의 단위 벡터
        hit.normal = normalize(hit.point - sphere.center)
    end if

    return hit
}

    3. 학습자 구현의 훌륭한 점 및 개선 제안

    훌륭한 점 (Key Achivements)

    1. 정밀한 성능 최적화의 이해: 방향 벡터 ray.dir이 단위 벡터일 때, 이차방정식의 \(a\) 값(\(\mathbf{D} \cdot \mathbf{D}\))을 \(1.0\)으로 생략할 수 있음을 주석으로 명시하고 식을 단순화했습니다. 이는 불필요한 내적(Dot product) 연산을 줄이는 훌륭한 그래픽스 최적화 기법입니다.
    2. 법선 벡터의 정규화(Normalization): 충돌 지점으로부터 구의 중심을 빼서 구한 법선 벡터를 glm::normalize를 통해 단위 벡터로 변환했습니다. 법선 벡터는 이후 퐁 반사 모델(Phong Reflection Model)이나 물리 기반 렌더링(PBR)에서 광학 계산의 정밀도를 위해 반드시 크기가 1인 단위 벡터여야 합니다.

    피드백 및 개선 제안 (Expert Review)

    1. 용어 정의의 수정 (nabla \(\to\) discriminant): 코드 내에서 판별식을 nabla로 명명했습니다. 수학에서 나블라(\(\nabla\))는 미분 연산자(Gradient 등)를 뜻하므로, 이차방정식의 판별식을 의미하는 기호인 델타(\(\Delta\))를 따서 delta 혹은 discriminant로 변수명을 변경하는 것이 가독성과 수학적 엄밀성에 더 좋습니다.
    2. 음수 영역(\(t < 0\)) 예외 처리의 부재: min(d1, d2)는 두 교점 중 단순히 더 작은 값을 선택합니다. 만약 광선의 시작점(카메라)이 구체 내부에 있거나, 구체가 카메라 뒤쪽에 있는 경우 d1 또는 d2가 음수가 될 수 있습니다. 카메라 뒤쪽의 충돌을 방지하기 위해 유효한 최소 거리(\(t_{min} \approx 10^{-4}\))보다 큰 최소 양의 실근을 선택하는 예외 처리가 필요합니다.
    // 개선안 예시
float d = d2; // d2가 일반적으로 더 가까운 앞쪽 교점 (-b/2 - sqrt)
if (d < 1e-4f) { // d2가 뒤에 있다면 d1을 확인
    d = d1;
    if (d < 1e-4f) return hit; // 둘 다 카메라 뒤에 있으면 충돌 없음
}
hit.d = d;

    4. WebGPU 인터랙티브 데모로의 확장 시각화

    이 C++ CPU 레이트레이서 알고리즘을 브라우저 환경의 WebGPU Compute Shader로 변환하면 실시간으로 60FPS 이상의 인터랙티브 데모를 빌드할 수 있습니다.

    시각화 방식

    1. Compute Shader (WGSL): 각 화면의 픽셀 좌표마다 1개의 워크그룹 스레드를 할당하고, C++ 코드와 동일한 광선-구체 교차 수식을 실행합니다.
    2. Normal-to-RGB 매핑 (시각화의 핵심): 충돌 지점에서 구한 법선 벡터 hit.normal은 \([-1.0, 1.0]\) 범위의 \((x, y, z)\) 값을 가집니다. 이를 화면에 직접 출력하기 위해 \([0.0, 1.0]\) 범위의 RGB 색상 공간으로 매핑합니다. $$ \text{Color}_{RGB} = \text{normal} \times 0.5 + 0.5 $$ 이를 통해 구체의 입체감이 무지개 빛깔의 부드러운 그라데이션으로 화면에 아름답게 표현됩니다.
    3. 인터랙션: 웹브라우저 상에서 마우스 드래그를 통해 구체의 위치(center)를 실시간으로 이동시키거나 반지름(radius)을 조절하며, 광선과 구체가 실시간으로 교차 판정을 일으키는 모습을 렌더링 화면으로 즉각 감상할 수 있습니다.
