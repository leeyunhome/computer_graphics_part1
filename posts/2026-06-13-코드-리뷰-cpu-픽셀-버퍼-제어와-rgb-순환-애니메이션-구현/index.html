    [코드 리뷰] CPU 픽셀 버퍼 제어와 RGB 순환 애니메이션 구현

    학습자가 DirectX 11 환경에서 CPU-GPU 간의 데이터 흐름을 이해하고, 화면을 구성하는 가장 기본 단위인 '픽셀(Pixel)'을 직접 제어해 보는 훌륭한 첫걸음을 내디뎠습니다.

    이번 커밋은 프레임 레이트를 인위적으로 조절하여 픽셀의 변화 과정을 관찰하고, 수학적 규칙(모듈러 연산)을 적용해 색상을 주기적으로 변화시키는 애니메이션의 기초를 다루고 있습니다.

    학습자가 직접 구현하며 이해한 핵심 개념

    1. 시간 지연(Sleep)을 통한 프레임 제어 및 디버깅

    학습자는 초당 수십~수백 번 실행되는 그래픽스 루프(Update)의 속도가 너무 빨라 픽셀이 채워지는 과정을 시각적으로 확인하기 어렵다는 문제를 인식했습니다. 이를 해결하기 위해 Sleep(300)을 도입하여 300ms(0.3초)마다 한 번씩 프레임이 갱신되도록 인위적인 지연을 주었습니다. 이는 실시간 렌더링 환경에서 시간 흐름에 따른 상태 변화를 시각적으로 추적(Debugging)하는 유용한 방법입니다.

    2. 모듈러 연산(\(\pmod\))을 활용한 상태 순환 (RGB Cycling)

    하드코딩된 단일 색상(빨간색)을 칠하던 기존 방식에서 벗어나, 프레임이 진행됨에 따라 빨강(R) \(\rightarrow\) 초록(G) \(\rightarrow\) 파랑(B) 순서로 색상이 계속 순환하는 규칙을 구현했습니다. 인덱스 \(i\)를 3으로 나눈 나머지를 활용하여 추가적인 조건문 누적 없이 간단한 수식만으로 상태 패턴을 제어하는 컴퓨터 과학의 기본기를 그래픽스에 잘 녹여냈습니다.

    3. 1차원 배열을 이용한 2차원 화면 버퍼 제어

    컴퓨터 그래픽스에서 화면(Canvas)은 2차원 그리드 형태이지만, 메모리 상에서는 1차원 배열(std::vector<Vec4> pixels)로 선형적으로 정렬되어 있습니다. 학습자는 1차원 인덱스 \(i\)를 증가시키며 픽셀을 순차적으로 채워나감으로써, 메모리 공간이 화면에 매핑되는 구조를 직관적으로 이해했습니다.

    핵심 알고리즘 추상화 (Pseudo-code)

    학습자가 작성한 업데이트 루프의 핵심 논리는 아래와 같이 추상화할 수 있습니다. 매 프레임마다 화면을 초기화하고, 시간에 따라 전진하는 인덱스 위치에 순환하는 RGB 색상을 하나씩 기록하는 방식입니다.

    function Update()
{
    // 1. 프레임 간 격차를 시각적으로 인지하기 위한 대기
    Wait(300 milliseconds)

    // 2. 2차원 화면 크기만큼의 픽셀 버퍼를 기본 배경색(밝은 회색)으로 초기화
    PixelBuffer = AllocateArray(Width * Height, DefaultColor)

    // 3. 현재 프레임 인덱스(i)에 해당하는 픽셀의 색상을 순환 규칙에 따라 결정
    ColorIndex = CurrentFrameIndex % 3

    if ColorIndex == 0 then
        PixelBuffer[CurrentFrameIndex] = Red
    else if ColorIndex == 1 then
        PixelBuffer[CurrentFrameIndex] = Green
    else
        PixelBuffer[CurrentFrameIndex] = Blue

    // 4. 다음 프레임을 위해 인덱스 증가 및 경계 검사
    CurrentFrameIndex = CurrentFrameIndex + 1
    if CurrentFrameIndex >= TotalPixels then
        CurrentFrameIndex = 0
}

    수식적 이해

    1. 모듈러 연산을 이용한 색상 결정식

    픽셀 인덱스 \(i\)에 따른 RGB 색상 벡터 \(\mathbf{C}(i)\)의 결정 공식은 다음과 같습니다.

    \[ \mathbf{C}(i) = \begin{cases} (1.0, 0.0, 0.0, 1.0) & \text{if } i \pmod 3 = 0 \quad (\text{Red}) \\ (0.0, 1.0, 0.0, 1.0) & \text{if } i \pmod 3 = 1 \quad (\text{Green}) \\ (0.0, 0.0, 1.0, 1.0) & \text{if } i \pmod 3 = 2 \quad (\text{Blue}) \end{cases} \]

    2. 1차원 인덱스의 2차원 좌표 매핑

    화면의 가로 너비를 \(W\)라 할 때, 학습자가 칠하고 있는 \(i\)번째 픽셀의 실제 화면상 2차원 좌표 \((x, y)\)는 다음과 같은 수학적 사상을 가집니다.

    \[ x = i \pmod W, \quad y = \lfloor \frac{i}{W} \rfloor \]

    이를 통해 인덱스 \(i\)가 증가함에 따라 픽셀이 왼쪽에서 오른쪽으로 채워지고, 한 줄을 다 채우면 아래 줄로 내려가는 주사(Scan) 방식의 원리를 수학적으로 유도할 수 있습니다.

    WebGPU 인터랙티브 데모

    이 학습 내용을 모던 웹 그래픽스 API인 WebGPU의 Compute Shader로 옮겨 시각화한다면 다음과 같이 훨씬 빠르고 동적인 병렬 데모를 만들 수 있습니다.

    데모 시각화 시나리오

    • 작동 방식: CPU에서 Sleep으로 스레드를 멈추는 대신, GPU에 시간 정보(Time Uniform)를 전달합니다.
    • Compute Shader의 병렬 처리:
      • 수만 개의 GPU 스레드가 동시에 화면의 모든 픽셀 좌표를 계산합니다.
      • 시간(\(t\))이 흐름에 따라 활성화되는 픽셀의 경계선 \(I_{active} = \lfloor t \times \text{Speed} \rfloor\)를 계산합니다.
      • 자신의 픽셀 인덱스 \(i\)가 \(I_{active}\)보다 작거나 같으면 학습자의 공식(\(i \pmod 3\))에 따라 빨강, 초록, 파랑 색상을 띠고, 그렇지 않은 영역은 회색으로 남아 점진적으로 화면이 RGB 패턴으로 물드는 애니메이션이 초당 60프레임 이상으로 부드럽게 연출됩니다.
    • 시각적 효과: 웹 브라우저 상에서 마우스 클릭이나 슬라이더를 통해 RGB가 채워지는 속도(Delay)를 실시간으로 조절하며, CPU 기반 소프트웨어 렌더링과 GPU 기반 병렬 렌더링의 성능 차이를 직관적으로 비교 체험할 수 있습니다.
