    1차원 버퍼와 프레임 업데이트를 통한 픽셀 애니메이션 구현

    학습자는 이번 커밋을 통해 CPU 측에서 정의된 2차원 래스터(Raster) 그래픽스 데이터를 1차원 연속 메모리 버퍼로 관리하고, 매 프레임마다 상태를 갱신하여 화면에 움직이는 애니메이션을 표현하는 핵심 그래픽스 기초 개념을 학습했습니다.

    1. 학습자가 직접 구현하며 이해한 핵심 개념

    (1) 2차원 공간의 1차원 선형 메모리 매핑

    컴퓨터 그래픽스에서 화면(Canvas)은 2차원 그리드(\(Width \times Height\)) 형태를 가집니다. 하지만 비디오 메모리(VRAM)나 시스템 메모리(RAM)는 기본적으로 1차원 배열 구조입니다. 학습자는 이를 매핑하는 공식을 이해하고 적용했습니다.

    2차원 좌표 \((x, y)\)를 1차원 인덱스 \(index\)로 변환하는 공식은 다음과 같습니다. $\(index = x + (y \times Width)\)$

    반대로, 1차원 인덱스 \(i\)로부터 현재 픽셀의 2차원 좌표 \((x, y)\)를 역산하면 다음과 같습니다. $\(x = i \pmod{Width}\)$ $\(y = \lfloor \frac{i}{Width} \rfloor\)$

    학습자는 인덱스 \(i\)를 매 프레임 순차적으로 증가시킴으로써, 빨간색 픽셀이 좌측 상단 \((0, 0)\)에서 시작하여 우측으로 이동하다가 화면 끝에 도달하면 아래 행의 첫 번째 열로 줄바꿈(Line-wrap)하며 이동하는 주사(Scanning) 방식을 코드로 구현해 냈습니다.

    (2) 안전성 및 방어적 프로그래밍 (Type Safety & Defensive Programming)

    이번 변경사항의 핵심은 부호 있는 정수형(int)에서 부호 없는 정수형(size_t)으로의 타입 변환과 경계 검사 조건식의 개선입니다.

    • 타입 안정성 (size_t): std::vector::size()는 컨테이너의 크기를 나타내는 부호 없는 정수형(size_t)을 반환합니다. 이전 코드의 int i와 비교할 경우 암묵적 형변환 및 서명 비교 경고(Signed/Unsigned mismatch warning)가 발생할 수 있습니다. 이를 size_t로 통일하여 타입 안전성을 높였습니다.
    • 방어적 경계 조건 (>=): 기존의 i == pixels.size() - 1 조건은 완벽히 순차적인 흐름에서는 작동하지만, 멀티스레딩이나 프레임 드롭, 혹은 수치 오버플로우 등의 예외 상황에서 i가 경계값을 초과해 버릴 경우 무한 루프나 유효하지 않은 메모리 참조(Out of Bounds) 문제를 발생시킬 수 있습니다. 이를 >= 비교 연산으로 변경함으로써 버퍼 오버플로우를 원천 차단하는 방어적 프로그래밍을 실천했습니다.

    2. 프레임 애니메이션 알고리즘 흐름 (의사코드)

    매 프레임 호출되는 렌더링 루프 내부에서 전체 캔버스를 초기화하고, 특정 픽셀의 위치를 변경하는 핵심 알고리즘은 다음과 같이 추상화할 수 있습니다.

    Function UpdateFrame():
    // 1. 상태 유지 변수 (정적 변수로 프레임간 상태 공유)
    Static Variable current_pixel_index = 0

    // 2. 캔버스 버퍼 초기화 (모든 픽셀을 회색으로 채움)
    Define Canvas = Array of size (Width * Height) with Color(0.8, 0.8, 0.8)

    // 3. 특정 위치의 픽셀 색상을 빨간색으로 변경
    Canvas[current_pixel_index] = Color(1.0, 0.0, 0.0)

    // 4. 다음 프레임을 위한 인덱스 증가 및 경계 검사 (방어적 초기화)
    current_pixel_index = current_pixel_index + 1
    If current_pixel_index >= Canvas.Length - 1 Then
        current_pixel_index = 0
    EndIf

    // 5. 변경된 버퍼를 그래픽 카드(VRAM)에 업로드 및 화면 출력
    UploadToGPU(Canvas)

    3. WebGPU 인터랙티브 데모

    이 개념을 웹 브라우저 표준 저수준 그래픽스 API인 WebGPU 환경에서 구현한다면, CPU가 아닌 GPU의 병렬 컴퓨트 셰이더(Compute Shader) 혹은 프래그먼트 셰이더(Fragment Shader)를 통해 훨씬 더 효율적으로 시각화할 수 있습니다.

    // WebGPU WGSL (WebGPU Shading Language) 예시 컨셉

struct Uniforms {
    canvas_size : vec2<f32>,
    frame_index : u32,
}
@group(0) @binding(0) var<uniform> ubo : Uniforms;

@fragment
fn fs_main(@builtin(position) frag_coord : vec4<f32>) -> @location(0) vec4<f32> {
    // 2차원 프래그먼트 좌표를 1차원 인덱스로 변환
    let x = u32(frag_coord.x);
    let y = u32(frag_coord.y);
    let width = u32(ubo.canvas_size.x);
    let current_index = x + (y * width);

    // CPU에서 매 프레임 증가하여 전달된 frame_index와 일치하는 픽셀만 빨간색으로 그림
    let target_index = ubo.frame_index % (u32(ubo.canvas_size.x * ubo.canvas_size.y));

    if (current_index == target_index) {
        return vec4<f32>(1.0, 0.0, 0.0, 1.0); // Red
    }
    return vec4<f32>(0.8, 0.8, 0.8, 1.0); // Background Gray
}

    브라우저에서의 시각화 모습

    1. 정적인 배경 속 움직임: 웹 페이지 내 생성된 캔버스(Canvas) 위에 수만 개의 회색 픽셀들이 격자 모양으로 배치됩니다.
    2. 고속 래스터 스캔: CPU가 매 프레임 증가시키는 frame_index를 전달받아, 단 하나의 빨간색 픽셀이 마치 브라운관 TV의 전자총이 화면을 훑고 지나가듯 좌측 상단에서 우측 하단 방향으로 빠르게 흘러갑니다.
    3. 무한 루프: 마지막 픽셀에 도달하는 순간, 지연 없이 다시 첫 번째 픽셀(좌측 상단 0번)로 돌아와 애니메이션이 부드럽게 반복됩니다.
