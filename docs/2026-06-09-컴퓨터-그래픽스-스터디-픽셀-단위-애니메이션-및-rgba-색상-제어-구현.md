# 컴퓨터 그래픽스 스터디: 픽셀 단위 애니메이션 및 RGBA 색상 제어 구현

안녕하세요! 컴퓨터 그래픽스 스터디 중 진행한 흥미로운 구현 내용을 공유하고자 합니다. 이번 커밋에서는 기본적인 캔버스에 픽셀 단위로 색상을 입히고, 이를 애니메이션하는 기능을 추가했습니다. 특히 RGB 색상을 순환하며 표현하고, 애니메이션 속도를 제어하는 부분에 중점을 두었습니다.

## ✨ 구현 목표 및 개요

이번 커밋의 핵심 목표는 다음과 같습니다:
1.  **픽셀 애니메이션:** 캔버스 상의 특정 픽셀을 순차적으로 업데이트하여 움직이는 듯한 효과를 만듭니다.
2.  **RGB 색상 순환:** 애니메이션되는 픽셀의 색상을 빨강(Red), 초록(Green), 파랑(Blue)으로 번갈아 가며 나타냅니다.
3.  **애니메이션 속도 제어:** 특정 딜레이를 주어 애니메이션의 진행 속도를 조절합니다.

이를 통해 가장 기본적인 수준에서 화면의 픽셀을 직접 제어하고, 시간의 흐름에 따라 시각적인 변화를 만들어내는 컴퓨터 그래픽스의 핵심 원리를 이해하고자 했습니다.

## 🚀 핵심 구현 내용

`Update()` 함수는 매 프레임마다 호출되어 화면을 갱신하는 역할을 합니다. 이번 구현에서는 이 `Update()` 함수 내에서 캔버스의 픽셀 배열을 직접 조작하여 애니메이션 효과를 만들어냈습니다.

### 1. 픽셀 배열과 캔버스 (Frame Buffer Abstraction)

우리 프로그램의 캔버스는 `Vec4` 타입의 요소를 가지는 벡터로 추상화됩니다. 여기서 `Vec4`는 $(R, G, B, A)$ 즉, 빨강, 초록, 파랑, 알파(투명도) 채널을 각각 0.0에서 1.0 사이의 값으로 표현하는 픽셀 색상입니다. 이 벡터는 실제 GPU의 프레임 버퍼와 유사하게 작동하며, 화면에 그려질 모든 픽셀의 색상 정보를 담고 있습니다.

```cpp
// 개념적인 픽셀 배열 초기화
std::vector<Vec4> pixels(canvasWidth * canvasHeight, Vec4{0.8f, 0.8f, 0.8f, 1.0f});
// 모든 픽셀을 회색으로 초기화 (R=0.8, G=0.8, B=0.8, A=1.0)
```

### 2. RGB 색상 순환 애니메이션

가장 흥미로운 부분은 픽셀의 색상을 시간에 따라 변경하는 방식입니다. `Update()` 함수가 호출될 때마다 `i`라는 인덱스 변수가 증가하며, 이 `i` 값에 따라 현재 업데이트될 픽셀의 색상을 결정합니다.

```cpp
// 의사 코드: 픽셀 색상 할당 로직
pixel_index = current_frame_index;

if (pixel_index % 3 == 0) {
    pixels[pixel_index] = Vec4{1.0f, 0.0f, 0.0f, 1.0f}; // 빨강
} else if (pixel_index % 3 == 1) {
    pixels[pixel_index] = Vec4{0.0f, 1.0f, 0.0f, 1.0f}; // 초록
} else { // pixel_index % 3 == 2
    pixels[pixel_index] = Vec4{0.0f, 0.0f, 1.0f, 1.0f}; // 파랑
}

// 다음 프레임을 위해 인덱스 증가
current_frame_index++;
// (캔버스 크기를 넘어가면 초기화)
```

여기서 `pixel_index % 3`이라는 **모듈러 연산 (Modular Arithmetic)**이 핵심적인 역할을 합니다. 이 연산을 통해 인덱스가 0, 1, 2, 0, 1, 2... 와 같이 3가지 값으로 반복되며, 결과적으로 픽셀의 색상이 빨강, 초록, 파랑으로 순환하도록 만듭니다. 이는 자원이나 색상 팔레트가 제한적인 환경에서 패턴을 만들거나 색상을 반복할 때 유용하게 사용될 수 있는 기법입니다.

### 3. 애니메이션 속도 제어 (Fixed Time Step)

애니메이션의 속도를 제어하기 위해 `Update()` 함수의 시작 부분에 300ms의 지연을 추가했습니다.

```cpp
// 의사 코드: 애니메이션 프레임 간 지연
Delay(300); // 300 밀리초 동안 실행을 일시 중단
```

이 `Delay(300)` 함수는 각 프레임 갱신 사이에 300밀리초(0.3초) 동안 프로그램의 실행을 멈춥니다. 이는 매우 간단하게 애니메이션의 진행 속도를 느리게 만드는 방법이지만, 동시에 몇 가지 중요한 학습 포인트를 제공합니다.

*   **장점:** 구현이 매우 간단합니다.
*   **단점:**
    *   **블로킹 (Blocking):** 이 함수가 실행되는 동안 프로그램의 메인 스레드가 완전히 멈춥니다. 즉, 다른 어떤 작업도 수행할 수 없게 됩니다. 이는 사용자 인터페이스의 응답성 저하로 이어질 수 있습니다.
    *   **프레임 레이트 종속성:** 이 방식은 시스템의 처리 능력과 무관하게 고정된 지연 시간을 부여합니다. 이는 특정 하드웨어 환경에서는 문제가 없을 수 있지만, 다양한 환경에서 일관된 애니메이션 속도를 보장하기 어렵습니다.
    *   **`delta time`의 필요성:** 실제 게임이나 그래픽스 애플리케이션에서는 `Sleep` 대신 `delta time` (이전 프레임으로부터 경과한 시간)을 사용하여 물리 연산이나 애니메이션을 프레임 레이트에 독립적으로 만듭니다.

## 💡 학습 내용 및 시사점

이번 구현을 통해 다음과 같은 중요한 컴퓨터 그래픽스 개념을 직접 경험하고 이해할 수 있었습니다.

*   **프레임 버퍼의 기초:** `std::vector<Vec4>`가 어떻게 가상의 프레임 버퍼 역할을 하는지, 그리고 픽셀 단위로 색상 정보를 저장하고 관리하는 방식을 이해했습니다. 이는 GPU가 화면에 그림을 그리는 방식에 대한 직관적인 이해를 돕습니다.
*   **RGBA 색상 모델:** $(R, G, B, A)$ 각 채널이 어떻게 색상과 투명도를 표현하는지 명확히 파악했습니다. 특히 0.0부터 1.0 사이의 부동 소수점 값을 사용하는 표준적인 접근 방식을 적용했습니다.
*   **애니메이션의 기본 원리:** `Update` 루프와 인덱스 증가를 통해 시간에 따라 시각적인 변화를 만들어내는 가장 기본적인 애니메이션 메커니즘을 구현했습니다. 이는 모든 복잡한 애니메이션의 근간이 됩니다.
*   **모듈러 연산의 활용:** `i % N`과 같은 간단한 수학 연산이 어떻게 반복적인 패턴이나 순환적인 효과를 만드는데 유용하게 사용될 수 있는지 배웠습니다.
*   **프레임 레이트 제어의 중요성:** `Sleep()` 함수를 사용하여 애니메이션 속도를 제어하는 단순한 방법을 시도했지만, 그 한계점(블로킹, 프레임 레이트 종속성)을 파악하며 더 정교한 `delta time` 기반의 프레임 레이트 제어 방식의 필요성을 깨달았습니다.
