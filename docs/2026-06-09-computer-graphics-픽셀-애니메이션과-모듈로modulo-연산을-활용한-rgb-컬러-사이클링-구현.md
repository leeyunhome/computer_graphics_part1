# [Computer Graphics] 픽셀 애니메이션과 모듈로(Modulo) 연산을 활용한 RGB 컬러 사이클링 구현

안녕하세요! 컴퓨터 그래픽스 기초를 다지는 과정의 일환으로, DirectX 11 환경에서 캔버스의 픽셀을 직접 제어하고 렌더링 루프를 이해하는 실습을 진행했습니다. 

이번 포스팅에서는 1차원 배열로 구성된 픽셀 버퍼에 애니메이션 효과를 적용하고, 수학적 연산을 활용해 RGB 색상을 순환(Color Cycling)시키는 기법에 대해 정리해 보겠습니다.

---

## 1. 구현 목표 및 핵심 개념

이번 실습의 주요 목표는 캔버스 상의 픽셀들을 순차적으로 순회하며 색상을 변경하는 **픽셀 애니메이션**을 구현하는 것입니다. 이를 위해 다음과 같은 컴퓨터 그래픽스의 기초 개념들을 활용했습니다.

### 1.1. 1차원 버퍼를 활용한 2D 캔버스 제어
우리가 보는 모니터 화면은 2차원 공간($Width \times Height$)이지만, 컴퓨터 메모리 상에서 이 픽셀 데이터들은 연속적인 **1차원 배열(Array)** 로 관리됩니다. 
특정 인덱스 $i$를 $0$부터 순차적으로 증가시키면 캔버스의 좌측 상단 픽셀부터 우측 하단 픽셀까지 차례대로 접근할 수 있습니다. 

### 1.2. RGBA 색상 공간과 벡터
컴퓨터 그래픽스에서 픽셀의 색상은 주로 4차원 벡터로 표현됩니다. 
$$ Color = (R, G, B, A) $$
각 요소는 Red, Green, Blue, Alpha(투명도)를 의미하며, $0.0$에서 $1.0$ 사이의 부동소수점(Float) 값으로 정규화하여 사용합니다. 예를 들어, 순수 빨간색은 $(1.0, 0.0, 0.0, 1.0)$으로 표현됩니다.

## 2. 핵심 구현 기법

### 2.1. 모듈로(Modulo) 연산을 이용한 컬러 사이클링 (Color Cycling)
단순히 한 가지 색상으로 픽셀을 칠하는 것을 넘어, 픽셀의 인덱스 위치에 따라 빨강(Red) $\rightarrow$ 초록(Green) $\rightarrow$ 파랑(Blue) 순으로 색상이 반복되도록 구현했습니다.

이러한 주기적인 패턴을 만들 때 가장 유용하게 사용되는 수학적 기법이 바로 **나머지 연산(Modulo)** 입니다. 현재 픽셀의 인덱스를 $i$라고 할 때, 색상을 결정하는 함수는 다음과 같이 정의할 수 있습니다.

$$
Color(i) =
\begin{cases}
(1.0, 0.0, 0.0, 1.0) & \text{if } i \equiv 0 \pmod 3 \\
(0.0, 1.0, 0.0, 1.0) & \text{if } i \equiv 1 \pmod 3 \\
(0.0, 0.0, 1.0, 1.0) & \text{if } i \equiv 2 \pmod 3
\end{cases}
$$

이러한 수학적 접근을 통해 별도의 복잡한 상태 변수(State variable) 없이, 인덱스 값 하나만으로 무한히 반복되는 다채로운 색상 패턴을 생성할 수 있습니다.

### 2.2. 프레임 지연을 통한 애니메이션 속도 제어
현대의 컴퓨터는 매우 빠르기 때문에, 렌더링 루프(Update Loop)가 1초에 수천 번 이상 실행될 수 있습니다. 만약 업데이트 속도를 제어하지 않으면 픽셀이 이동하는 과정이 사람의 눈에 보이지 않고 순식간에 지나가 버리게 됩니다.

따라서 렌더링 루프 내에 **인위적인 지연(Delay)** 을 주어 애니메이션이 사람의 시각에 인식될 수 있도록 속도를 조절했습니다. *(참고: 실제 상용 게임 엔진에서는 스레드를 멈추는 대신 델타 타임($\Delta t$)을 계산하여 프레임 독립적인 애니메이션을 구현하지만, 기초 실습 단계에서는 직관적인 제어를 위해 슬립(Sleep) 방식을 활용했습니다.)*

## 3. 개념적 의사코드 (Pseudo-code)

실제 작성한 코드를 추상화하여, 어떤 논리적 흐름으로 픽셀 애니메이션이 동작하는지 의사코드로 정리했습니다.

```cpp
// 매 프레임마다 호출되는 업데이트 루프
function UpdateRendering():
    // 1. 애니메이션 속도 조절 (예: 300ms 대기)
    Wait(300)
    
    // 2. 전체 캔버스를 기본 배경색으로 초기화
    InitializeBuffer(BackgroundColor)
    
    // 3. 모듈로 연산을 통한 RGB 컬러 사이클링 적용
    if currentIndex % 3 == 0:
        SetPixel(currentIndex, RED)
    else if currentIndex % 3 == 1:
        SetPixel(currentIndex, GREEN)
    else:
        SetPixel(currentIndex, BLUE)
        
    // 4. 다음 픽셀로 인덱스 이동
    currentIndex = currentIndex + 1
    
    // 5. 캔버스의 끝에 도달하면 처음으로 되돌림 (Loop)
    if currentIndex >= TotalPixels:
        currentIndex = 0
```

## 4. 배운 점 및 마무리

이번 커밋을 통해 다음과 같은 중요한 그래픽스 기초를 확립할 수 있었습니다.

* **메모리와 픽셀의 관계:** 1차원 벡터 배열의 데이터를 조작하는 것이 최종적으로 모니터의 2D 픽셀에 어떻게 매핑되는지 직관적으로 이해했습니다.
* **수학의 시각화:** 프로그래밍에서 자주 쓰이는 나머지(Modulo) 연산이 그래픽스에서는 패턴과 반복을 만들어내는 강력한 도구로 활용된다는 점을 실습을 통해 확인했습니다.
* **렌더링 루프와 시간 제어:** 눈에 보이는 렌더링 결과를 만들기 위해서는 하드웨어의 처리 속도와 인간의 시각적 인지 속도 사이의 간극을 조절(Time Control)하는 과정이 필수적임을 배웠습니다.

단순히 픽셀 하나를 움직이는 작은 구현이지만, 이는 향후 다루게 될 프래그먼트 셰이더(Fragment Shader)나 파티클 시스템을 이해하는 튼튼한 밑거름이 될 것입니다. 다음 단계에서는 본격적으로 DirectX의 파이프라인을 활용한 렌더링 기법들을 학습해 나갈 예정입니다. 🚀
