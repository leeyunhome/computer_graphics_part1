# 픽셀 애니메이션 구현과 프레임 기반 갱신 원리

안녕하세요! DirectX 11 / C++ 그래픽스 스터디에 오신 것을 환영합니다.
"Animate red pixel moving across canvas frame by frame" 커밋 메시지와 코드 변경사항을 통해 학습자님이 픽셀 기반 애니메이션의 기본적인 원리를 성공적으로 이해하고 구현한 것을 확인할 수 있습니다.

## 학습자가 직접 구현하며 이해한 개념

이번 커밋을 통해 학습자님은 다음과 같은 핵심적인 그래픽스 개념들을 직접 구현하며 체득했을 것으로 보입니다.

1.  **프레임 기반 애니메이션 (Frame-based Animation) 원리**:
    `Update()` 함수가 매 프레임(또는 매 게임 틱)마다 호출될 때마다 화면의 내용을 갱신하는 방식을 이해했습니다. `static int i` 변수를 사용하여 이전 프레임의 상태(픽셀 위치)를 기억하고, 매 프레임마다 그 상태를 변경함으로써 시간에 따라 움직이는 효과를 만들어냈습니다. 이는 모든 실시간 그래픽스 애플리케이션의 기본이 되는 개념입니다.

2.  **픽셀 버퍼 직접 조작 및 렌더링 파이프라인 이해**:
    `std::vector<Vec4> pixels`라는 픽셀 데이터를 저장하는 버퍼를 직접 초기화하고, 특정 픽셀의 색상을 변경하는 방식을 사용했습니다. 이는 DirectX 11의 텍스처 업데이트 파이프라인(`D3D11_MAPPED_SUBRESOURCE ms`)으로 이어져, CPU에서 변경된 픽셀 데이터가 GPU로 전송되어 화면에 최종적으로 렌더링되는 과정을 이해하는 데 큰 도움이 됩니다. 이전에는 미리 정해진 픽셀을 출력했다면, 이제는 프로그램 로직에 따라 동적으로 픽셀 색상을 변경하고 이를 렌더링 파이프라인에 반영하는 방법을 익힌 것입니다.

3.  **1차원 배열에서 2D 캔버스 픽셀 위치 관리**:
    `pixels` 벡터는 실제로는 2차원 캔버스의 모든 픽셀을 1차원 배열 형태로 저장하고 있습니다. 학습자님은 `i`라는 단일 인덱스를 사용하여 캔버스의 모든 픽셀을 순회하며 빨간색 픽셀을 이동시켰습니다. 이는 2차원 좌표 $(x, y)$를 1차원 배열의 인덱스로 매핑하는 기본적인 방법을 implicitly하게 활용한 것입니다.
    $$ \text{index} = y \times \text{canvasWidth} + x $$
    현재 코드에서는 `i` 자체가 0부터 `canvasWidth * canvasHeight - 1`까지 순차적으로 증가하므로, 모든 픽셀을 가로로 스캔하며 이동하는 것과 동일한 효과를 냅니다.

4.  **순환 애니메이션 구현**:
    `if (i == pixels.size() - 1)` 조건문을 통해 픽셀이 캔버스의 끝에 도달하면 다시 시작점으로 돌아오도록 구현했습니다. 이는 애니메이션이 끊임없이 반복되도록 하는 기본적인 순환 로직입니다.

## 핵심 알고리즘 (의사코드)

학습자님이 `Update()` 함수에서 구현한 핵심 알고리즘은 다음과 같이 의사코드로 설명할 수 있습니다.

```
함수 Update():
    // 1. 현재 프레임의 픽셀 위치 추적을 위한 정적 변수 'i' 초기화 (최초 1회만)
    static int i = 0

    // 2. 캔버스 크기만큼의 픽셀 버퍼 생성 및 모든 픽셀을 기본색(회색)으로 초기화
    std::vector<Vec4> pixels (canvasWidth * canvasHeight)를 회색으로 채움

    // 3. 현재 픽셀 위치 'i'에 해당하는 픽셀을 빨간색으로 설정
    pixels[i] = 빨간색 (Vec4{ 1.0f, 0.0f, 0.0f, 1.0f })

    // 4. 다음 프레임을 위한 픽셀 위치 'i' 업데이트
    만약 'i'가 픽셀 버퍼의 마지막 인덱스(pixels.size() - 1)와 같다면:
        'i'를 0으로 재설정 (처음으로 돌아감)
    그렇지 않다면:
        'i'를 1 증가시킴

    // 5. 업데이트된 픽셀 데이터를 GPU 텍스처 버퍼로 전송
    D3D11_MAPPED_SUBRESOURCE를 사용하여 텍스처 버퍼를 매핑하고,
    'pixels' 벡터의 데이터를 해당 버퍼로 복사
    텍스처 버퍼 언매핑
```

## WebGPU 인터랙티브 데모

만약 이 픽셀 애니메이션을 브라우저 기반의 WebGPU Compute Shader로 시각화한다면, 다음과 같은 방식으로 구현되고 인터랙티브하게 표현될 수 있습니다.

1.  **Compute Shader의 병렬 처리**: WebGPU Compute Shader는 수많은 스레드를 병렬로 실행하여 대량의 데이터를 효율적으로 처리하는 데 특화되어 있습니다. 이 데모에서는 각 픽셀 하나하나가 독립적인 컴퓨트 스레드에 의해 처리되는 것처럼 동작할 수 있습니다.

2.  **데이터 버퍼**: 캔버스의 모든 픽셀 색상 정보를 담는 `storage buffer` (또는 `texture_storage_2d`)를 WebGPU GPU 메모리에 할당합니다. Compute Shader는 이 버퍼에 직접 접근하여 픽셀 색상을 읽고 쓸 수 있습니다.

3.  **애니메이션 로직**:
    *   Compute Shader는 `global_invocation_id`를 사용하여 현재 스레드가 어떤 픽셀을 처리해야 하는지 식별합니다.
    *   `uniform buffer`를 통해 현재 프레임 번호나 경과 시간과 같은 애니메이션 상태 변수를 Compute Shader에 전달합니다.
    *   Compute Shader 내에서, 전달받은 프레임 번호를 기반으로 빨간색 픽셀이 위치해야 할 `target_pixel_index`를 계산합니다. 이 계산은 C++ 코드에서 `i`를 업데이트하는 로직과 유사할 것입니다.
    *   모든 픽셀에 대해, 만약 현재 스레드의 `global_invocation_id`가 `target_pixel_index`와 일치한다면 해당 픽셀을 빨간색으로 설정하고, 그렇지 않다면 회색으로 설정합니다.
    *   이 과정을 매 프레임마다 Compute Shader를 디스패치(dispatch)하여 반복하면, 빨간색 픽셀이 캔버스 전체를 순환하며 움직이는 애니메이션이 부드럽게 재생됩니다.

4.  **렌더링**: Compute Shader가 모든 픽셀 색상을 계산하여 `storage buffer`에 저장하면, 이 버퍼의 데이터를 `texture`로 사용하여 WebGPU 렌더링 파이프라인 (Vertex Shader, Fragment Shader)에서 최종적으로 화면에 그려냅니다.

**인터랙티브 요소**: WebGPU 데모에서는 단순히 픽셀을 움직이는 것을 넘어, 마우스 클릭이나 키보드 입력에 따라 픽셀의 속도를 조절하거나, 이동 경로를 변경하거나, 여러 개의 픽셀이 동시에 움직이도록 확장할 수 있습니다. 예를 들어, 마우스 클릭 위치를 빨간색 픽셀의 목표 지점으로 설정하여 쫓아가게 만들 수도 있습니다. Compute Shader는 이러한 복잡한 픽셀 단위 연산을 매우 효율적으로 수행할 수 있는 강력한 도구입니다.
