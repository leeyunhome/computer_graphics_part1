# ImGui를 활용한 캔버스 배경색 제어 구현

학습자님, DirectX 11 환경에서 ImGui를 이용해 캔버스 배경색을 실시간으로 제어하는 기능을 성공적으로 구현하셨습니다. 이번 커밋은 그래픽스 애플리케이션에서 UI와 렌더링 로직을 연동하는 중요한 단계를 잘 보여줍니다.

## 핵심 구현 내용

이번 커밋의 핵심은 ImGui의 컬러 슬라이더 위젯 (`ImGui::SliderFloat3`)과 DirectX 11 캔버스의 배경색을 연결하여, 사용자가 UI를 통해 렌더링 결과를 즉시 확인할 수 있도록 만든 것입니다.

1.  **ImGui UI 활성화**: `main.cpp`에서 주석 처리되었던 ImGui 초기화 및 프레임 생성 관련 코드(`ImGui_ImplDX11_NewFrame`, `ImGui::NewFrame`, `ImGui::Render` 등)를 다시 활성화하여 ImGui 인터페이스가 화면에 나타나도록 했습니다.
2.  **배경색 변수 연결**: `main.cpp`에서 ImGui의 `SliderFloat3` 위젯이 `example` 객체의 `backgroundColor` 멤버 변수를 직접 조작하도록 변경했습니다. (diff에는 `Example` 클래스에 `backgroundColor` 멤버 변수를 추가한 내용은 없지만, 구현의 맥락상 추가되었음을 예상할 수 있습니다.)
3.  **캔버스 배경색 적용**: `Example.h`의 `Update()` 함수에서 캔버스 픽셀 버퍼(`std::vector<Vec4> pixels`)를 초기화할 때, 하드코딩된 특정 색상 값(`Vec4{0.8f, 0.8f, 0.8f, 1.0f}`) 대신 `backgroundColor` 멤버 변수의 값을 사용하도록 수정했습니다. 이로써 ImGui 슬라이더로 변경된 색상이 다음 프레임에 캔버스 배경색으로 반영됩니다.
4.  **불필요한 지연 제거**: `Update()` 함수 내에 있던 `Sleep(300)` 호출을 주석 처리하여, 매 프레임 발생하는 불필요한 대기 시간을 제거하고 애플리케이션의 반응성을 높였습니다. 이는 일반적으로 개발 초기에 디버깅 목적으로 넣는 코드이지만, 실제 동작에서는 제거하는 것이 좋습니다.
5.  **픽셀 애니메이션 주석 처리**: 기존에 특정 픽셀의 색상을 순차적으로 변경하던 애니메이션 코드를 주석 처리하여, 이번 구현의 핵심인 "배경색 제어" 기능에 집중하도록 했습니다.

## 학습자가 직접 구현하며 이해한 개념

이번 구현을 통해 학습자님은 다음과 같은 핵심 개념들을 직접 경험하고 이해하셨을 것으로 보입니다.

1.  **ImGui와 DirectX 11 렌더링 파이프라인 연동**:
    *   ImGui가 별도의 렌더링 백엔드(`ImGui_ImplDX11_RenderDrawData`)를 통해 자체 UI 요소를 그리는 방식을 이해했습니다.
    *   일반적인 렌더링 루프에서 ImGui의 프레임 시작(`ImGui::NewFrame`)과 렌더링 명령 기록(`ImGui::Render`)이 주 렌더링(`example->Render()`) 전후로 어떻게 위치해야 하는지를 파악했습니다. 특히, ImGui의 드로우 데이터를 DX11 커맨드 리스트에 넘겨주는 과정(`ImGui_ImplDX11_RenderDrawData`)이 스왑 체인(`example->swapChain->Present`) 호출 전에 이루어져야 함을 알게 되었습니다.

2.  **데이터 흐름 관리**:
    *   UI 요소(ImGui 슬라이더)의 변경 사항이 애플리케이션의 핵심 로직(`Example` 클래스의 `backgroundColor`)에 어떻게 전달되고, 그 값이 렌더링 결과(`pixels` 버퍼 초기화)에 어떻게 반영되는지에 대한 전체적인 데이터 흐름을 구축했습니다.
    *   이는 다음과 같은 순서로 진행됩니다.
        *   ImGui 슬라이더 조작 -> `example->backgroundColor` 값 변경
        *   `example->Update()` 호출 -> `backgroundColor` 값을 사용해 캔버스 픽셀 버퍼 `pixels` 초기화
        *   `pixels` 데이터를 DirectX 텍스처로 업로드
        *   `example->Render()` 호출 -> 텍스처를 화면에 렌더링

3.  **캔버스 픽셀 버퍼 초기화 및 업데이트**:
    *   CPU 메모리에 `std::vector<Vec4>` 형태로 캔버스 전체 픽셀 데이터를 준비하고, 이 데이터를 GPU 텍스처로 업로드하는 방식을 구현했습니다.
    *   특정 색상으로 전체 캔버스를 채우는 과정은 다음과 같이 추상화할 수 있습니다:
        ```cpp
        // Example::Update()
        std::vector<Vec4> pixels(canvasWidth * canvasHeight);
        Vec4 currentBgColor = { backgroundColor[0], backgroundColor[1], backgroundColor[2], 1.0f };

        for (int y = 0; y < canvasHeight; ++y)
        {
            for (int x = 0; x < canvasWidth; ++x)
            {
                pixels[y * canvasWidth + x] = currentBgColor;
            }
        }
        // ... (pixels 데이터를 D3D11_MAPPED_SUBRESOURCE를 통해 GPU 텍스처로 복사)
        ```
    *   여기서 캔버스 픽셀을 나타내는 각 색상 값은 $ \mathbf{C} = (R, G, B, A) $ 형태의 4차원 벡터로 표현됩니다.

4.  **DirectX 11 리소스 매핑(Mapping)**:
    *   `D3D11_MAPPED_SUBRESOURCE` 구조체와 `deviceContext->Map()`/`Unmap()` 함수를 사용하여 CPU 메모리(`pixels` 데이터)의 내용을 GPU 리소스(텍스처)로 효율적으로 전송하는 방식을 이해했습니다. 이는 GPU가 렌더링에 사용할 수 있도록 CPU 데이터를 GPU 메모리에 복사하는 중요한 과정입니다.

## 개선점 및 추가 고려사항

현재 구현은 기능적으로 완벽하지만, 앞으로 더 발전시킬 수 있는 몇 가지 지점들을 제안합니다.

1.  **객체 지향적 접근**: 현재 `main.cpp`에서 `example->backgroundColor` 멤버 변수에 직접 접근하고 있습니다. 이는 `Example` 클래스의 캡슐화를 저해할 수 있습니다. 대신 `Example` 클래스 내에 `SetBackgroundColor(float r, float g, float b)`와 같은 `public` 메서드를 만들고, ImGui 슬라이더는 이 메서드를 호출하여 배경색을 변경하도록 하는 것이 더 바람직합니다.
    ```cpp
    // Example.h (클래스 내부)
    public:
        void SetBackgroundColor(float r, float g, float b) {
            backgroundColor[0] = r;
            backgroundColor[1] = g;
            backgroundColor[2] = b;
        }

    // main.cpp
    ImGui::SliderFloat3("RGB(0.0->1.0)", example->backgroundColor, 0.0f, 1.0f); // -> 이렇게 변경
    // float tempColor[3] = { example->backgroundColor[0], example->backgroundColor[1], example->backgroundColor[2] };
    // if (ImGui::SliderFloat3("RGB(0.0->1.0)", tempColor, 0.0f, 1.0f)) {
    //     example->SetBackgroundColor(tempColor[0], tempColor[1], tempColor[2]);
    // }
    ```
    이 방식은 `backgroundColor`의 내부 구현이 변경되어도 `main.cpp` 코드를 수정할 필요가 없게 만들어 유지보수성을 높여줍니다.

2.  **성능 최적화**: 현재 `Update()` 함수는 매 프레임 `std::vector<Vec4> pixels`를 새로 생성하고 모든 픽셀을 배경색으로 채웁니다. 배경색이 실제로 변경될 때만 이 무거운 작업을 수행하도록 최적화할 수 있습니다. 예를 들어, 이전 프레임의 배경색과 현재 프레임의 배경색을 비교하여 다를 때만 픽셀 버퍼를 업데이트하는 플래그를 사용할 수 있습니다.

3.  **색상 공간 (Color Space)**: ImGui 슬라이더는 일반적으로 선형(Linear) RGB 값을 다루지만, 대부분의 모니터는 sRGB 색상 공간을 사용합니다. 나중에 감마 보정(Gamma Correction)을 렌더링 파이프라인에 적용하여, 사용자가 보는 색상이 더 정확하고 자연스럽게 보이도록 할 수 있습니다. 감마 보정은 일반적으로 픽셀의 색상 값 $ C $에 대해 $ C^{2.2} $를 계산하는 방식으로 이루어집니다.

## WebGPU 인터랙티브 데모

이번에 구현한 배경색 제어 기능을 WebGPU 컴퓨트 셰이더로 시각화한다면 다음과 같은 방식으로 인터랙티브 데모를 만들 수 있습니다.

1.  **WebGPU 설정**: 브라우저에서 `GPUAdapter`, `GPUDevice`, `GPUQueue` 등을 초기화하고, 캔버스와 렌더링 파이프라인을 설정합니다.
2.  **Uniform Buffer**: ImGui 슬라이더에서 조절된 배경색 (`vec4<f32>`)을 저장할 `GPUBuffer`를 생성하고, 이를 `uniform` 타입으로 바인딩합니다.
    *   Uniform Buffer 데이터: $ \{R, G, B, A\} $
3.  **Compute Shader 작성**: 각 픽셀의 색상을 계산하는 컴퓨트 셰이더를 작성합니다. 이 셰이더는 `uniform`으로 전달된 배경색 값을 읽어와 해당 픽셀의 색상으로 지정합니다.
    ```wgsl
    // background_color_shader.wgsl
    @group(0) @binding(0) var<uniform> backgroundColor: vec4<f32>;
    @group(0) @binding(1) var<storage, write> outputTexture: array<vec4<f32>>; // 캔버스 픽셀 데이터

    @compute
    fn main(@builtin(global_invocation_id) global_id: vec3<u32>) {
        let width = 캔버스_너비; // 상수 또는 다른 uniform으로 전달
        let index = global_id.y * width + global_id.x;
        outputTexture[index] = backgroundColor;
    }
    ```
4.  **컴퓨트 파이프라인 및 Dispatch**: 위 컴퓨트 셰이더를 사용하여 `GPUComputePipeline`을 생성하고, 캔버스 크기만큼의 작업 그룹을 디스패치합니다. 각 작업 그룹의 스레드는 특정 픽셀의 색상을 계산하게 됩니다.
5.  **렌더링**: 컴퓨트 셰이더에서 계산된 `outputTexture`의 데이터를 렌더 파이프라인으로 전달하여 최종적으로 캔버스에 그립니다.
6.  **ImGui 연동**: HTML/JavaScript 기반의 ImGui (또는 다른 UI 라이브러리)를 사용하여 색상 슬라이더를 만들고, 이 슬라이더의 값이 변경될 때마다 WebGPU `uniformBuffer`의 내용을 업데이트하고 컴퓨트 셰이더를 다시 디스패치하여 실시간으로 배경색이 바뀌는 모습을 볼 수 있습니다.

이처럼 WebGPU 컴퓨트 셰이더는 수많은 픽셀을 병렬로 처리하는 데 매우 효율적이며, UI를 통해 파라미터를 조작하여 실시간으로 렌더링 결과를 업데이트하는 데 강력한 도구가 됩니다.
