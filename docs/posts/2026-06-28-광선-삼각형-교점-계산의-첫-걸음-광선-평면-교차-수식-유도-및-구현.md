# 광선-삼각형 교점 계산의 첫 걸음: 광선-평면 교차 수식 유도 및 구현

이 커밋은 3D 컴퓨터 그래픽스 레이트레이싱의 가장 핵심적인 기초 중 하나인 **광선-삼각형 교점(Ray-Triangle Intersection) 검출** 알고리즘의 1단계(광선-평면 교차 계산)를 성공적으로 구현한 단계입니다. 

학습자는 단순한 API 호출을 넘어, 광선과 평면의 수학적 방정식을 결합하여 매개변수 $t$를 유도하는 과정을 코드로 직접 옮겼습니다.

---

## 1. 핵심 수학적 개념 및 수식 유도

삼각형은 하나의 3차원 평면 위에 존재합니다. 따라서 광선과 삼각형의 교점을 찾으려면 우선 **광선과 평면이 만나는 교점**을 찾은 뒤, 그 교점이 **삼각형의 세 변 안쪽에 존재하는지** 판별하는 2단계 전략을 취해야 합니다. 이번 커밋은 이 중 1단계를 구현한 것입니다.

### 광선과 평면의 방정식 결합

* **광선(Ray)의 방정식**: 시점 $o$(Origin)에서 방향 벡터 $d$(Direction)로 진행하는 광선 위의 임의의 점 $p$는 매개변수 $t$를 이용하여 다음과 같이 표현됩니다.
  $$ p(t) = o + td \quad (t \ge 0) $$

* **평면(Plane)의 방정식**: 평면 위의 한 점 $v_0$와 평면의 법선 벡터(Normal) $n$이 주어졌을 때, 평면 위의 임의의 점 $p$는 다음 관계식을 만족합니다. 평면 위의 임의의 벡터 $(p - v_0)$는 법선 벡터 $n$과 수직이기 때문입니다.
  $$ (p - v_0) \cdot n = 0 $$

### 매개변수 $t$의 유도 과정
광선의 식 $p(t)$를 평면의 방정식에 대입하여 $t$에 대해 정리합니다.
$$ ((o + td) - v_0) \cdot n = 0 $$
$$ (o - v_0) \cdot n + t(d \cdot n) = 0 $$
$$ t(d \cdot n) = (v_0 - o) \cdot n $$
$$ t = \frac{(v_0 - o) \cdot n}{d \cdot n} = \frac{v_0 \cdot n - o \cdot n}{d \cdot n} $$

학습자는 위 유도 과정을 바탕으로 코드에 아래 수식을 정확히 반영하였습니다.
```cpp
t = (dot(v0, faceNormal) - dot(orig, faceNormal)) / dot(dir, faceNormal);
```

## 2. 코드 분석 및 구현의 의미

단순히 수식을 대입하는 것에 그치지 않고, 수치 해석적 안정성과 그래픽스 렌더링 파이프라인의 효율성을 위한 예외 처리가 꼼꼼히 반영되어 있습니다.

1. **외적(Cross Product)을 통한 평면의 법선 벡터 계산**
   삼각형의 두 에지(Edge) 벡터 $v_1 - v_0$와 $v_2 - v_0$를 외적한 후 정규화하여 평면의 법선 벡터 $n$을 구합니다. 이 벡터는 교점 계산뿐만 아니라 추후 **퐁 반사 모델(Phong Reflection Model)**을 적용하여 광원 효과를 계산할 때 필수적인 정보가 됩니다.

2. **백페이스 컬링(Backface Culling) 처리**
   ```cpp
   if (dot(-dir, faceNormal) < 0.0f) return false;
   ```
   광선의 진행 방향 역벡터 `-dir`과 삼각형의 법선 벡터 `faceNormal` 사이의 사잇각이 $90^\circ$보다 크면(내적 값이 음수이면) 카메라를 등지고 있는 뒷면으로 판단하여 조기에 충돌 대상에서 제외합니다. 렌더링 성능 향상에 도움을 줍니다.

3. **제로 디비전(Division by Zero) 방지**
   ```cpp
   if (glm::abs(dot(dir, faceNormal)) < 1e-2f) return false;
   ```
   분모에 해당하는 $d \cdot n$이 $0$에 가깝다는 것은 광선과 평면이 거의 평행하다는 의미입니다. 이 경우 교점이 존재하지 않거나 무한대에 위치하므로, 임계값(`1e-2f`) 이하일 때는 충돌하지 않은 것으로 처리하여 부동소수점 오버플로우 오류를 예방했습니다.

4. **유효 거리 검증**
   ```cpp
   if (t < 0.0f) return false;
   ```
   수학적으로 평면과의 교점이 광선의 시작점 뒤쪽에 있는 경우(즉, 카메라 뒤편에 배치된 삼각형)를 걸러내어 불필요한 렌더링을 방지합니다.

## 3. 핵심 알고리즘 흐름 (의사코드)

학습자가 구현한 알고리즘의 추상화된 흐름은 다음과 같습니다.

```text
function IntersectRayTriangle(Ray ray, Triangle tri) {
    // 1. 삼각형 평면의 법선 벡터 계산
    Vector3 edge1 = tri.v1 - tri.v0;
    Vector3 edge2 = tri.v2 - tri.v0;
    Vector3 normal = Normalize(Cross(edge1, edge2));

    // 2. 예외 처리: 뒷면 거르기 & 평행 여부 검사
    if (Dot(-ray.dir, normal) < 0) return Miss;
    if (Absolute(Dot(ray.dir, normal)) < epsilon) return Miss;

    // 3. 광선-평면 교점 계산 (수식 대입)
    float t = (Dot(tri.v0, normal) - Dot(ray.origin, normal)) / Dot(ray.dir, normal);
    if (t < 0) return Miss; // 카메라 뒤쪽에 있는 평면

    Vector3 hitPoint = ray.origin + t * ray.dir;

    // 4. [Next Step] 충돌점(hitPoint)이 삼각형 영역 내부에 있는지 판단
    // (Barycentric Coordinates 또는 Inside-Outside Test 적용 예정)
    
    return Hit(t, hitPoint, normal);
}
```

## 4. WebGPU 인터랙티브 데모 시각화 제안

이 알고리즘을 최신 웹 표준인 **WebGPU**의 Compute Shader를 이용해 브라우저 상에서 시각화한다면 다음과 같은 훌륭한 인터랙티브 데모를 만들 수 있습니다.

* **동적 광선 시각화**: 마우스 커서의 움직임에 따라 카메라에서 나가는 광선(Ray)의 궤적을 3차원 공간 상에 실선으로 그립니다.
* **영역별 컬러 코딩 (Debug View)**:
  * **회색 (Gray)**: 광선이 평면과 평행하여 아예 교점이 생기지 않는 영역.
  * **반투명 주황색 (Orange)**: 광선-평면 교점($t$)은 성공적으로 구했으나, 아직 삼각형 내부 판정을 통과하지 못한 영역 (즉, 무한 평면과의 교점).
  * **빨간색 (Red)**: 최종적으로 삼각형 내부 판정까지 통과하여 완벽한 교점이 형성된 영역.
* **실시간 파라미터 제어**: UI 슬라이더를 통해 삼각형의 세 정점($v_0, v_1, v_2$)의 위치를 실시간으로 변경하며 평면 법선 벡터가 어떻게 변화하는지 웹 브라우저에서 실시간으로 관찰할 수 있습니다.
* **퐁 반사 모델(Phong Reflection Model) 적용**: 교점 검출이 완료되면 법선 벡터 `faceNormal`과 임의의 포인트 광원(Point Light)의 위치 정보를 셰이더로 넘겨주어, 교점에 Ambient, Diffuse, Specular 값이 합쳐진 입체감 있는 3D 조명 효과를 즉각적으로 렌더링해 볼 수 있습니다.
