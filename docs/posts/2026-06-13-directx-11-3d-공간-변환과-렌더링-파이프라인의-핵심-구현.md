# [DirectX 11] 3D 공간 변환과 렌더링 파이프라인의 핵심 구현

학습자가 이번 커밋("add source code")을 통해 구현한 핵심 코드는 **DirectX 11 API를 활용하여 3D 정점(Vertex) 데이터를 정의하고, 이를 GPU 렌더링 파이프라인을 거쳐 화면에 출력하는 기초 프레임워크**입니다. 

단순히 화면에 삼각형을 그리는 단계에서 나아가, 3차원 정점 정보가 수학적 변환을 거쳐 2D 스크린 공간에 투영되는 그래픽스 파이프라인의 전 과정을 직접 C++와 HLSL(High-Level Shader Language)로 구현하며 이해한 내용을 정리합니다.

---

## 1. 직접 구현하며 이해한 핵심 그래픽스 개념

### 1.1 CPU와 GPU의 통신: 상수 버퍼(Constant Buffer)
GPU는 한 번에 수많은 정점을 병렬 처리하지만, 모든 정점에 공통으로 적용되어야 하는 데이터(예: 변환 행렬, 빛의 위치 등)가 있습니다. 학습자는 이를 위해 CPU에서 GPU 메모리로 데이터를 전송하는 **상수 버퍼(Constant Buffer)**의 개념과 라이프사이클을 이해하고 구현했습니다.
* **`D3D11_USAGE_DYNAMIC` 및 `D3D11_MAP_WRITE_DISCARD`**: 매 프레임마다 변하는 행렬 정보를 CPU에서 GPU로 빠르게 업로드하기 위해 버퍼를 매핑(Mapping)하고 언매핑하는 일련의 과정을 제어했습니다.

### 1.2 3D 공간 변환 Pipeline (WVP)
3차원 상의 정점($Local$ 공간)이 모니터 화면($Screen$ 공간)에 그려지기까지는 총 4가지 주요 좌표계 변환을 거칩니다. 학습자는 정점 셰이더(Vertex Shader)에서 이 변환 연산을 수행하도록 설계했습니다.
1. **World 변환**: 로컬 모델 좌표계를 가상 세계(World)의 중심 기준으로 이동, 회전, 크기 변환합니다.
2. **View 변환**: 카메라를 기준으로 모든 오브젝트의 상대적 위치를 재정의합니다. (카메라의 월드 행렬의 역행렬)
3. **Projection 변환**: 원근감(Perspective)을 부여하고, 3D 공간을 NDC(Normalized Device Coordinates, $-1$에서 $1$ 사이의 표준화된 부피 공간)로 투영합니다.

## 2. 수학적 정량화: 공간 변환 공식

3차원 정점 $v_{local} = [x, y, z, 1]^T$ 행벡터가 최종 클립 공간(Clip Space)의 $v_{clip}$으로 변환되는 수학적 행렬 곱셈 연산은 다음과 같습니다. DirectX는 전통적으로 행벡터(Row-vector) 방식을 사용하므로 연산 순서는 좌측에서 우측으로 진행됩니다.

$$ v_{world} = v_{local} \cdot M_{world} $$

$$ v_{view} = v_{world} \cdot M_{view} $$

$$ v_{clip} = v_{view} \cdot M_{projection} $$

이를 단일 합성 행렬(WVP Matrix)로 묶어 정점 셰이더에서 단 한 번의 행렬 곱셈으로 최적화합니다.

$$ v_{clip} = v_{local} \cdot (M_{world} \cdot M_{view} \cdot M_{projection}) $$

이후 GPU 하드웨어는 클립 공간의 좌표를 $w$ 성분으로 나누는 **원근 분할(Perspective Divide)**을 자동으로 수행하여 최종 NDC 좌표를 얻습니다.

$$ v_{ndc} = \begin{bmatrix} x / w \\ y / w \\ z / w \end{bmatrix} $$

## 3. 핵심 알고리즘 및 파이프라인 제어 흐름 (의사코드)

학습자가 구현한 C++ 어플리케이션의 메인 루프 중 **렌더링 파이프라인 설정 및 상수 버퍼 갱신 과정**을 추상화한 의사코드입니다.

```cpp
// 1. 프레임 업데이트: 시간에 따른 오브젝트 회전 행렬 계산
void Game::Update(float deltaTime)
{
    // 로컬 축 기준으로 회전하는 월드 행렬 생성
    Matrix rotation = Matrix::CreateRotationY(accumulatedTime);
    Matrix translation = Matrix::CreateTranslation(0.0f, 0.0f, 2.0f);
    worldMatrix = rotation * translation;

    // View 및 Projection 행렬 갱신 (카메라 위치 고정)
    viewMatrix = Camera::GetViewMatrix();
    projectionMatrix = Camera::GetProjectionMatrix();
}

// 2. 프레임 렌더링: GPU 파이프라인에 바인딩 및 그리기
void Game::Render()
{
    // 2.1 CPU에서 계산한 WVP 행렬을 GPU Constant Buffer에 복사
    ConstantData cbData;
    cbData.worldViewProjection = (worldMatrix * viewMatrix * projectionMatrix).Transpose(); // HLSL 열우선 대응을 위한 전치

    context->Map(constantBuffer, 0, D3D11_MAP_WRITE_DISCARD, 0, &mappedResource);
    memcpy(mappedResource.pData, &cbData, sizeof(ConstantData));
    context->Unmap(constantBuffer, 0);

    // 2.2 디바이스 콘텍스트에 파이프라인 단계 설정
    context->IASetInputLayout(vertexLayout);
    context->IASetPrimitiveTopology(D3D11_PRIMITIVE_TOPOLOGY_TRIANGLELIST);
    
    // 버퍼 바인딩
    context->IASetVertexBuffers(0, 1, &vertexBuffer, &stride, &offset);
    context->VSSetConstantBuffers(0, 1, &constantBuffer);
    
    // 셰이더 활성화
    context->VSSetShader(vertexShader, nullptr, 0);
    context->PSSetShader(pixelShader, nullptr, 0);

    // 2.3 그리기 명령 제출
    context->Draw(vertexCount, 0);
}
```

## 4. WebGPU 인터랙티브 데모로 확장하기

DirectX 11에서 학습한 이 고정/프로그래머블 파이프라인 구조와 3D 변환 개념은 최신 웹 표준 그래픽스 API인 **WebGPU**에서도 거의 완벽하게 대칭되어 적용됩니다. 

이 프로젝트의 결과물을 브라우저상에서 작동하는 **WebGPU Compute & Render Shader 데모**로 변환한다면 다음과 같이 시각화할 수 있습니다.

```
+-----------------------------------------------------------+
| [ WebGPU Interactive 3D Space Visualizer ]                |
|                                                           |
|    ( 3D Cube / Particles )                                |
|          /\                                               |
|         /  \   <-- CPU(JS)에서 WGSL 셰이더로              |
|        /____\      uniform buffer를 통해 WVP 행렬 실시간 전달 |
|        |    |                                             |
|        |____|                                             |
|                                                           |
|  [Control Panel]                                          |
|  Rotation Speed: [===|======]  Camera FOV: [=====|====]   |
+-----------------------------------------------------------+
```

### 데모 작동 방식
1. **WGSL(WebGPU Shading Language)로 이식**: DX11의 HLSL 셰이더 코드는 WGSL의 `@vertex` 스테이지 코드로 변환되어 동작합니다. `binding(0)` 그룹의 `uniform` 버퍼를 통해 실시간으로 변환 행렬을 전달받습니다.
2. **Compute Shader의 활용**: 만약 수만 개의 3D 큐브를 개별 제어한다면, WebGPU의 Compute Shader를 사용하여 GPU 상에서 직접 각 큐브의 World 행렬을 물리 시뮬레이션으로 계산하고, 이를 스토리지 버퍼(Storage Buffer)에 저장한 뒤 렌더링 파이프라인으로 직접 전달(Zero-copy)하는 고성능 물리 효과 데모를 브라우저 내에서 가속기 인터랙션으로 구현할 수 있습니다.
