# 벡터 외적을 이용한 삼각형의 법선 벡터 및 면적 계산

이번 커밋은 레이트레이싱(Ray Tracing)에서 가장 기본적이면서도 중요한 기하학적 단위인 **삼각형(Triangle)**을 정의하고, 이를 통해 빛의 물리적 상호작용을 계산하기 위한 수학적 기초를 정립하는 단계입니다. 

학습자는 삼각형의 세 정점을 이용해 두 개의 변(Edge) 벡터를 생성하고, 이들의 **외적(Cross Product)**을 통해 평면의 법선 벡터(Normal Vector)와 삼각형의 면적(Area)을 도출하는 과정을 다이어그램과 문서로 정리하였습니다. 이는 레이트레이싱 렌더러에서 정밀한 충돌 판정(Ray-Triangle Intersection) 및 퐁 반사 모델(Phong Reflection Model)을 적용하기 위한 필수 관문입니다.

---

## 핵심 수학적 개념 및 물리적 의미

3차원 공간에서 세 점 $A, B, C$로 구성된 삼각형이 존재할 때, 이 삼각형이 마주하는 방향과 크기를 구하기 위해 벡터의 외적을 사용합니다.

```text
       C
      / \
e2   /   \
    /     \
   A-------B
      e1
```

### 1. 변 벡터(Edge Vectors)의 생성
삼각형의 세 정점 중 한 점 $A$를 시점으로 하여 나머지 두 점 $B, C$를 향하는 두 개의 변 벡터 $\vec{e_1}$과 $\vec{e_2}$를 정의합니다.
$$\vec{e_1} = B - A$$
$$\vec{e_2} = C - A$$

### 2. 외적(Cross Product)과 평면의 법선 벡터(Normal Vector)
두 변 벡터 $\vec{e_1}$과 $\vec{e_2}$를 외적하면, 두 벡터가 이루는 평면에 동시에 수직인 새로운 벡터 $\vec{w}$가 생성됩니다.
$$\vec{w} = \vec{e_1} \times \vec{e_2}$$

이 외적 결과물인 $\vec{w}$를 정규화(Normalization)하면 삼각형 표면의 방향을 나타내는 단위 법선 벡터 $\vec{N}$을 얻을 수 있습니다. 이 법선 벡터는 광선(Ray)이 삼각형 표면에 부딪혔을 때, 빛이 어느 방향으로 반사되는지 계산하는 **퐁 반사 모델(Phong Reflection Model)**의 핵심 입력값이 됩니다.
$$\vec{N} = \frac{\vec{e_1} \times \vec{e_2}}{\|\vec{e_1} \times \vec{e_2}\|}$$

### 3. 외적의 크기와 삼각형의 면적(Area)
두 벡터 외적의 크기(Magnitude) $\|\vec{e_1} \times \vec{e_2}\|$는 두 벡터를 변으로 하는 평행사변형의 넓이와 같습니다. 따라서 우리가 구하고자 하는 삼각형의 면적은 이 값의 정확히 절반이 됩니다.
$$\text{Area} = \frac{1}{2} \|\vec{e_1} \times \vec{e_2}\|$$

이 면적 계산법은 단순히 삼각형의 크기를 구하는 것을 넘어, 광선과 삼각형의 교점 위치를 나타내는 **무게중심 좌표계(Barycentric Coordinates)**를 구할 때 분모와 분자로 활용되어, 텍스처 좌표(UV)나 정점 색상을 보간하는 데 결정적인 역할을 합니다.

## 알고리즘 추상화 (Pseudo-code)

학습자가 이해한 수학적 흐름을 레이트레이싱의 기하학적 파이프라인으로 추상화하면 다음과 같습니다.

```cpp
struct Vertex {
    Vector3 position;
};

struct Triangle {
    Vertex v0, v1, v2;

    // 삼각형의 기하학적 특성을 계산하는 함수
    GeometryInfo CalculateGeometry() {
        // 1. 변 벡터 계산
        Vector3 edge1 = v1.position - v0.position;
        Vector3 edge2 = v2.position - v0.position;

        // 2. 외적 수행
        Vector3 crossProduct = Vector3::Cross(edge1, edge2);

        // 3. 외적 크기를 이용한 면적 계산
        float parallelogramArea = crossProduct.Length();
        float triangleArea = 0.5f * parallelogramArea;

        // 4. 외적 결과를 정규화하여 법선 벡터 추출
        Vector3 normal = crossProduct.Normalize();

        return GeometryInfo{ normal, triangleArea };
    }
};
```

## WebGPU 인터랙티브 데모 제안

이 개념을 웹 브라우저 환경에서 실시간으로 시각화하기 위해 **WebGPU Compute Shader**와 그래픽스 파이프라인을 활용한 인터랙티브 데모를 다음과 같이 구성할 수 있습니다.

```text
[ 화면 구성 안 ]
+------------------------------------------+
|  (WebGPU Canvas)                         |
|   * 마우스로 드래그 가능한 세 정점 A, B, C   |
|   * 실시간으로 변화하는 삼각형 표현          |
|   * 삼각형 중심에서 뻗어나오는 법선 벡터(N)   |
|                                          |
|  [실시간 데이터 출력]                       |
|  - Edge 1: (x, y, z)                     |
|  - Edge 2: (x, y, z)                     |
|  - Cross Product: (x, y, z)              |
|  - Triangle Area: 12.45                  |
+------------------------------------------+
```

### 작동 원리 및 시각화 요소

1. **정점의 실시간 제어**:
   사용자가 브라우저 화면에서 삼각형의 세 꼭짓점($A, B, C$)을 마우스 드래그로 움직이면, WebGPU의 Uniform Buffer를 통해 세 정점의 3D 좌표가 Compute Shader 및 Vertex Shader로 실시간 전달됩니다.

2. **Compute Shader의 역할**:
   GPU의 병렬 연산 장치 내에서 위의 외적 공식과 크기 공식을 실행합니다.
   * `cross(edge1, edge2)`를 계산하여 외적 벡터의 방향과 길이를 구합니다.
   * 계산된 면적 값과 정규화된 법선 벡터를 렌더링 파이프라인으로 전달합니다.

3. **시각적 피드백 (Visual Feedback)**:
   * **법선 벡터 시각화**: 삼각형의 무게중심에서 외적 방향으로 붉은색 화살표(Line List)를 그립니다. 화살표의 길이는 정규화된 벡터($\vec{N}$) 혹은 외적의 크기($\|\vec{e_1} \times \vec{e_2}\|$)에 비례하도록 설정하여 면적의 변화를 직관적으로 이해할 수 있게 합니다.
   * **면적의 동적 시각화**: 삼각형 내부를 외적의 크기(면적)에 비례하는 밝기나 색상(예: 면적이 넓어질수록 푸른빛이 강해짐)으로 채워, 기하학적 크기의 변화를 시각적으로 인지하도록 돕습니다.
   * **퐁 반사 모델(Phong Reflection Model) 실시간 적용**: 마우스 움직임에 따라 실시간으로 변하는 법선 벡터($\vec{N}$)와 가상의 광원(Light) 사이의 각도를 내적(Dot Product)하여, 삼각형 표면에 실시간 디퓨즈(Diffuse) 및 스펙큘러(Specular) 하이라이트가 맺히는 모습을 보여줍니다.
