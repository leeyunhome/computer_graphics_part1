# 1차원 배열을 통한 2D 캔버스 제어와 CPU-GPU 메모리 매핑

학습자가 DirectX 11 환경에서 2D 캔버스를 초기화하고, CPU에서 직접 픽셀 데이터를 제어하여 GPU의 텍스처 버퍼로 업로드하는 그래픽스 파이프라인의 기초를 성공적으로 구현했습니다. 

이번 커밋은 단순히 색상 하나를 추가한 것을 넘어, 2D 스크린 공간의 좌표계를 1차원 선형 메모리에 매핑하는 컴퓨터 그래픽스의 핵심 기초 개념을 직접 코드로 검증한 중요한 단계입니다.

---

## 학습자가 직접 구현하며 이해한 핵심 개념

### 1. 2D 격자 좌표의 1차원 선형 매핑 (Flattening 2D Coordinates)
컴퓨터 메모리는 본질적으로 1차원 주소 공간입니다. 화면에 보이는 2차원 이미지 $W \times H$를 메모리에 표현하기 위해서는 2차원 좌표 $(x, y)$를 1차원 인덱스 $Index$로 변환해야 합니다. 학습자는 다음과 같은 공식의 구조를 이해하고 코드로 구현하였습니다.

$$Index(x, y) = x + (Width \times y)$$

* **빨간색 픽셀 (0, 0):** `0 + canvasWidth * 0` $\rightarrow$ 메모리 0번째 원소
* **노란색 픽셀 (1, 0):** `1 + canvasWidth * 0` $\rightarrow$ 메모리 1번째 원소
* **파란색 픽셀 (2, 0):** `2 + canvasWidth * 0` $\rightarrow$ 메모리 2번째 원소 (이번 커밋에서 추가된 부분)

이 수식은 이미지 처리뿐만 아니라 GPU 렌더 타깃, 텍스처 샘플링 등 그래픽스 전반에 걸쳐 가장 기초가 되는 물리적 주소 계산 방식입니다.

### 2. RGBA 색상 공간의 부동소수점 표현
정밀한 그래픽스 연산을 위해 색상은 각 채널당 $0.0$(어두움)에서 $1.0$(가장 밝음) 사이의 실수(float) 값으로 표현됩니다. 학습자는 `Vec4` 구조체를 사용하여 알파(Alpha) 채널을 포함한 RGBA 색상을 정의했습니다.
* 파란색 (Blue): `Vec4{ 0.0f, 0.0f, 1.0f, 1.0f }` (R=0, G=0, B=1, A=1)

---

## 핵심 알고리즘 및 데이터 흐름 (추상화)

CPU에서 생성된 픽셀 배열이 어떻게 GPU가 사용하는 텍스처 버퍼로 전송되는지 추상화된 흐름으로 설명합니다.

### 픽셀 버퍼 생성 및 업데이트 알고리즘

```cpp
// 1. CPU 시스템 메모리에 2D 이미지를 위한 1차원 픽셀 배열 확보
Array<Color4D> CPU_PixelBuffer(CanvasWidth * CanvasHeight, GrayColor);

// 2. 특정 좌표 (x, y)에 직접 픽셀 색상 기록
CPU_PixelBuffer[0 + CanvasWidth * 0] = Red;
CPU_PixelBuffer[1 + CanvasWidth * 0] = Yellow;
CPU_PixelBuffer[2 + CanvasWidth * 0] = Blue; // 학습자가 추가한 파란색 픽셀

// 3. CPU 메모리 데이터를 GPU 텍스처 리소스로 복사 (Map-Unmap 패턴)
GpuResource MappedBuffer = GPU_Context.Map(GPU_Texture2D, WriteDiscard);
CopyMemory(MappedBuffer.Data, CPU_PixelBuffer.Pointer, TotalSizeInBytes);
GPU_Context.Unmap(GPU_Texture2D);
```

이 방식은 CPU가 직접 프레임 버퍼를 그리고 이를 GPU에 매 프레임 업로드하는 **소프트웨어 래스터라이저(Software Rasterizer)**의 구조적 기틀이 됩니다.

## WebGPU 인터랙티브 데모

이 개념을 현대적인 웹 표준인 **WebGPU**의 **Compute Shader**를 활용하여 브라우저에서 시각화한다면 다음과 같이 작동합니다.

### 시각화 모습
* 브라우저 화면에 연한 회색($0.8, 0.8, 0.8$)으로 가득 찬 가로 $W$ x 세로 $H$ 크기의 캔버스가 렌더링됩니다.
* 캔버스의 왼쪽 가장 상단 맨 왼쪽 구석(좌표 $0,0$, $1,0$, $2,0$)에 돋보기로 확대해 보아야 할 정도로 작은 **빨간색(Red), 노란색(Yellow), 파란색(Blue)**의 세 개의 픽셀이 가로로 나란히 빛나고 있는 모습을 관찰할 수 있습니다.

### WebGPU 셰이더 내부 동작 원리
WebGPU의 Compute Shader에서는 CPU에서 1차원 배열을 일일이 계산하여 올리는 대신, GPU 내부의 수많은 스레드가 병렬로 각자의 픽셀 좌표를 계산하여 쓰기 가능한 텍스처(`texture_storage_2d`)에 직접 접근합니다.

```rust
// WGSL (WebGPU Shading Language) Compute Shader 개념도
@group(0) @binding(0) var outputTexture: texture_storage_2d<rgba8unorm, write>;

@compute @workgroup_size(1, 1)
fn main(@builtin(global_invocation_id) id: vec3<u32>) {
    let coords = id.xy; // 각 스레드가 담당하는 2D 픽셀 좌표 (x, y)

    var color = vec4<f32>(0.8, 0.8, 0.8, 1.0); // 기본 배경색 (회색)

    if (coords.y == 0u) {
        if (coords.x == 0u) { color = vec4<f32>(1.0, 0.0, 0.0, 1.0); } // Red
        if (coords.x == 1u) { color = vec4<f32>(1.0, 1.0, 0.0, 1.0); } // Yellow
        if (coords.x == 2u) { color = vec4<f32>(0.0, 0.0, 1.0, 1.0); } // Blue
    }

    textureStore(outputTexture, coords, color);
}
```

이 데모는 CPU 기반의 순차적 픽셀 연산과 GPU 기반의 병렬 픽셀 연산이 어떻게 동일한 수학적 기반($2D \leftrightarrow 1D$ 변환) 위에서 이루어지는지 직관적으로 보여줍니다.
