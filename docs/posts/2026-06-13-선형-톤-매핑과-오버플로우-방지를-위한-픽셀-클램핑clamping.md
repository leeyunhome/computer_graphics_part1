# 선형 톤 매핑과 오버플로우 방지를 위한 픽셀 클램핑(Clamping)

이번 커밋은 이미지의 전체적인 밝기(Brightness)를 선형적으로 증가시키면서, LDR(Low Dynamic Range) 표현 범위를 벗어나는 픽셀 데이터의 왜곡(Overexposure/Whiteout)을 방지하기 위해 **클램핑(Clamping)** 기법을 적용한 핵심적인 그래픽스 기초 구현을 담고 있습니다.

---

## 1. 학습자가 직접 구현하며 이해한 핵심 개념

### 선형 밝기 조절 (Linear Brightness Scaling)
이미지의 밝기를 조절하는 가장 직관적인 방법은 각 픽셀의 RGB 채널 값에 일정한 스칼라 값을 곱하는 것입니다. 학습자는 기존의 배율 $1.2$에서 $1.5$로 밝기 계수(Brightness Factor)를 상향하여 더 밝은 결과를 얻고자 했습니다.

### 오버플로우(Overflow)와 화이트아웃(Whiteout) 방지
컴퓨터 그래픽스에서 표준 모니터(LDR Display)가 표현할 수 있는 색상의 범위는 각 채널당 $[0.0, 1.0]$ (또는 정수형 8비트 기준 $0 \sim 255$)로 제한됩니다. 
* 기존 코드처럼 단순히 $1.2$를 곱하기만 하면, 원본 값이 $0.85$ 이상인 픽셀은 연산 결과가 $1.0$을 초과하게 됩니다.
* 이 상태로 그래픽 파이프라인이나 이미지 포맷에 데이터를 전달하면, 시스템에 따라 데이터 오버플로우가 발생하여 엉뚱한 색상으로 왜곡되거나 강제로 잘려 나가는 현상이 발생합니다.
* 학습자는 `std::clamp`를 도입하여 모든 RGB 채널의 최종 값이 $[0.0, 1.0]$ 범위 내에 안전하게 머물도록 보장했습니다. 이를 통해 밝은 영역이 완전히 깨지지 않고 하얗게 포화(Saturate)되도록 제어했습니다.

---

## 2. 핵심 수학 공식

이 변환은 픽셀 단위의 **선형 톤 매핑(Linear Tone Mapping)**의 가장 단순한 형태입니다. 입력 색상 벡터를 $\mathbf{C}_{in} = (R_{in}, G_{in}, B_{in})$, 밝기 조절 계수를 $s$, 최종 출력 색상 벡터를 $\mathbf{C}_{out}$이라고 할 때, 수학적 표현은 다음과 같습니다.

$$ \mathbf{C}_{out} = \max\left( \mathbf{0}, \min\left( \mathbf{1}, s \cdot \mathbf{C}_{in} \right) \right) $$

각 채널별로 전개하면 다음과 같은 독립적인 클램핑 연산이 수행됩니다.

$$ R_{out} = \text{clamp}(R_{in} \times s, 0.0, 1.0) $$
$$ G_{out} = \text{clamp}(G_{in} \times s, 0.0, 1.0) $$
$$ B_{out} = \text{clamp}(B_{in} \times s, 0.0, 1.0) $$

---

## 3. 핵심 알고리즘 (의사코드)

학습자가 C++ 이미지 루프 내에서 구현한 픽셀 병렬 연산 구조는 다음과 같이 추상화할 수 있습니다. 이 연산은 GPU의 픽셀 셰이더(Pixel Shader)나 컴퓨트 셰이더(Compute Shader)가 병렬로 처리하는 방식과 개념적으로 동일합니다.

```text
function AdjustBrightnessAndClamp(Image inputImage, float factor):
    Image outputImage = CreateEmptyImage(inputImage.width, inputImage.height)
    
    for each pixel in inputImage:
        // 1. 원본 RGB 색상 가져오기
        Color originalColor = pixel.color 
        
        // 2. 선형 스케일링 적용 및 범위 제한 [0.0, 1.0]
        Color processedColor;
        processedColor.r = Clamp(originalColor.r * factor, 0.0, 1.0)
        processedColor.g = Clamp(originalColor.g * factor, 0.0, 1.0)
        processedColor.b = Clamp(originalColor.b * factor, 0.0, 1.0)
        
        // 3. 결과 이미지에 저장
        outputImage[pixel.index] = processedColor
        
    return outputImage
```

## 4. 그래픽스 전문가로서의 분석 및 피드백

### 긍정적인 평가
* **예외 처리의 중요성 인지**: 단순히 곱셈 연산만 수행하던 기존 코드에서 발생할 수 있는 잠재적 그래픽 아티팩트(색상 깨짐 등)를 방지하기 위해 `std::clamp`를 선제적으로 적용한 점은 매우 훌륭합니다.
* **톤 매핑의 시작점**: 고대비 이미지나 HDR 환경에서 밝기를 다룰 때 왜 제한(Constraint)이 필요한지 직접 코드로 겪어보며 톤 매핑의 기초를 잘 다졌습니다.

### 한 단계 나아가기 (Deep Dive)
현재 적용한 방식은 **선형 클램핑(Linear Clamping)**입니다. 이 방식은 $1.0$을 초과하는 밝은 영역의 디테일이 모두 평평한 백색($1.0$)으로 뭉개지는 **클리핑(Clipping)** 현상을 유발합니다. 또한, RGB 채널 중 하나만 먼저 $1.0$에 도달하면 원본의 색상 비율이 깨져 색조가 왜곡(Color Shift)될 수 있습니다.

추후 이 개념을 확장하여, 아주 밝은 영역도 부드럽게 감쇄시키며 대비를 보존하는 **비선형 톤 매핑 알고리즘**(예: Reinhard, ACES Film Tone Mapping)을 공부하고 적용해 보는 것을 추천합니다.

## 5. WebGPU 인터랙티브 데모

이 연산을 브라우저 환경에서 GPU를 사용해 실시간으로 구현하면 어떻게 시각화할 수 있을까요? **WebGPU Compute Shader**를 활용한 가상의 인터랙티브 데모 구조는 다음과 같습니다.

### 시각화 구성 요소
1. **입력 캔버스**: 원본 이미지(`colosseum.jpg`)를 WebGPU 텍스처로 업로드하여 화면 좌측에 렌더링합니다.
2. **조절 슬라이더**: 사용자가 실시간으로 밝기 계수($s$)를 `1.0`부터 `3.0`까지 조절할 수 있는 UI 슬라이더를 제공합니다.
3. **출력 캔버스**: 컴퓨트 셰이더가 실시간으로 연산한 결과(`result.png`)를 화면 우측에 초당 60프레임으로 출력합니다.

### WebGPU WGSL 컴퓨트 셰이더 코드 예시
WebGPU에서 각 픽셀을 병렬로 처리하여 밝기를 조절하고 클램핑하는 WGSL 코드는 다음과 같은 형태로 구현됩니다.

```wgsl
@group(0) @binding(0) var inputTexture: texture_2d<f32>;
@group(0) @binding(1) var outputTexture: texture_storage_2d<rgba8unorm, write>;
@group(0) @binding(2) var<uniform> brightnessFactor: f32;

@compute @workgroup_size(16, 16)
fn main(@builtin(global_invocation_id) GlobalID: vec3<u32>) {
    let texSize = textureDimensions(inputTexture);
    let coords = vec2<i32>(GlobalID.xy);

    // 텍스처 경계 외부 접근 방지
    if (coords.x >= i32(texSize.x) || coords.y >= i32(texSize.y)) {
        return;
    }

    // 1. 원본 픽셀 읽기
    let rawColor = textureLoad(inputTexture, coords, 0);

    // 2. 밝기 스케일링 및 [0.0, 1.0] 범위로 클램핑 (WGSL 내장 함수 clamp 사용)
    let scaledRGB = clamp(rawColor.rgb * brightnessFactor, vec3<f32>(0.0), vec3<f32>(1.0));

    // 3. 결과 저장 (알파 채널은 원본 유지)
    textureStore(outputTexture, coords, vec4<f32>(scaledRGB, rawColor.a));
}
```

### 데모 작동 효과
* 사용자가 슬라이더를 오른쪽으로 밀어 `brightnessFactor`를 $1.5$, $2.0$으로 올리면 콜로세움의 어두운 그늘진 벽면이 실시간으로 환하게 밝아집니다.
* 동시에 하늘 영역처럼 이미 충분히 밝은 곳은 왜곡된 노이즈 패턴 없이 자연스럽고 하얗게 포화(Saturate)되는 모습을 눈으로 직접 확인할 수 있습니다.
